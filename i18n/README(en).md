## ✨ Features

### ✨ Experience & Navigation Engine

* **UI Refactoring**: Redesigned interface based on Windows File Explorer for a superior operating experience.
* **Turbo Mode**: Replaces web-end logic and blocks native sync (**Recommended to enable in Settings**).
* **Advanced Path Bar**: Supports scroll wheel navigation, dropdown switching, path echoing, and source backtracking.
* **Background Indexing**: A blue breathing dot on the home icon indicates the directory tree sync status.
* **Enhanced UX**: Supports Dark Mode, video duration display, one-click Blur Cover Images (Privacy Mode), Starred items, and file type sorting.

> 1. **Preparation**: Install the browser extension [Tampermonkey](https://www.tampermonkey.net/).

## ❓ FAQ

**Q: Why is the blue floating ball not appearing after installing the script?**
**A:** Ensure you are logged into the PikPak web end. If it still doesn't appear, try refreshing the page (F5) or check if it's being blocked by ad-blockers (like AdGuard / uBlock Origin). Try disabling the blocker or whitelisting the site.

**Q: What is "Turbo Mode" and why is it recommended?**
**A:** This is the script's core "black tech." When enabled, it physically blocks the memory-intensive native database sync logic and replaces it with the script's lightweight management. Advantage: It completely fixes lag or browser crashes (OOM) after minutes of use, allowing drives with tens of thousands of files to run as smoothly as desktop software.

**Q: Why does "My Pack" on the home page show a "Default" tag and prevent Stars, Copy, Move, Rename, or Delete?**
**A:** This is an official safety mechanism. The script has removed the share restrictions for "My Pack" on top of this protection.

**Q: What is the difference between "File Deduplication" and "Folder Deduplication"?**
**A:** 1. **File Deduplication**: Targets "individual files." Uses hash exact matching, video duration similarity, and name similarity. Best for cleaning scattered duplicate files. 2. **Folder Deduplication**: Targets the "entire folder structure." Uses name match, internal file overlap (Similarity Match), and subset redundancy (Containment Match). It can "see through" folders; it is perfect for cleaning duplicate TV series or manga packs even if folder names differ.

> **Advice**: Use "File Deduplication" for loose files and "Folder Deduplication" for large resource packs. Use hash matching in File Analysis for safe one-click deletion. Similarity algorithms should be reviewed manually to avoid accidental deletion.

**Q: Why does "Image Search" sometimes ask me to paste instead of uploading directly?**
**A:** This is due to browser security policies (CORS). When the script cannot push the image directly to the search server, it copies a screenshot of the current frame to your clipboard. Simply press `[Ctrl+V]` on the opened search page.

**Q: Why is there sound but no video when playing certain files?**
**A:** This usually happens with m3u8 multi-track streams where audio and video tracks are separated after cloud transcoding, which the web player cannot parse. We recommend clicking "External Play" to use professional players like PotPlayer.

**Q: Why are recent operations (Upload/Delete) from the Mobile App not syncing in the script's analysis list?**
**A:** To ensure maximum speed, the script uses a memory snapshot mechanism. If you have modified many files on other clients, please refresh the webpage to re-fetch the Global Index. Avoid simultaneous modifications across multiple devices during script operations.

**Q: Can I set different passwords for multiple files during Bulk Extract?**
**A:** Yes. Smart-matching logic: Save known passwords in the "Password Vault." During bulk extraction, the script automatically tries all passwords in the vault for every archive. Successful matches will start automatically.

**Q: What if Bulk Extract fails?**
**A:** Check if the correct password is in the Password Vault. If the file itself is corrupted, extraction will also fail on the official server side.

**Q: What information is contained in the exported JSON config?**
**A:** The JSON contains your local preferences, Blacklist/Whitelist rules, and Password Vault data. It **does not** contain your PikPak account credentials; it is safe for cross-device migration.

**Q: Why are some files "Protected" during bulk deletion?**
**A:** Check if these files are recorded in the Resource Manager. If "Skip resources recorded in manager on delete" is enabled in settings, the script protects these items. Solution: 1. Disable the protection in settings; or 2. Use "Run Cleanup Now" within the Resource Manager to force deletion.

## 🛡️ Privacy & Security

* **Local Data**: All core functions (Bulk Rename, File Analysis, Extraction, etc.) interact directly with PikPak APIs via your browser. Your Token, Vault, and file data are **saved only in your local browser**.
* **No Collection**: The script **does not** collect user data and **never** uploads your file info or credentials to third-party servers.
* **Third-party Interfaces**: Only extension features like "Online Subtitle Search" or "Image Search" send necessary keywords or image features to public service APIs. No personal identity information is involved.

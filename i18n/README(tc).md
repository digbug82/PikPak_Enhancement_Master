## 主要功能

### 體驗與導航增強

* **側鍵導覽**：支援滑鼠側鍵前進 / 後退，可在不同目錄層級間快速切換。
* **進階路徑列**：支援滾輪滑動、同層目錄下拉切換、路徑回顯與溯源跳轉。
* **全盤搜尋增強**：支援按檔名與檔案路徑搜尋，關鍵字局部標紅，長檔名會優先顯示命中關鍵字附近內容。
* **瀏覽偏好**：支援排序偏好、視圖偏好、保持瀏覽位置、媒體模式、隱藏按鈕文字、模糊媒體封面縮圖與夜間模式。

### 連結收藏夾與雲歸檔

* **連結收藏夾**：連結收藏夾同時儲存書籤連結、腳本同步資料、雲歸檔磁力連結與雲設定。
* **雲歸檔**：支援將所選檔案中的可追溯磁力連結歸檔到「連結收藏夾 > 雲歸檔」，可刪除來源檔案以節省空間。
* **雲歸檔恢復解析**：已歸檔磁力連結可重新提交雲下載任務，並回寫提交狀態。
* **雲歸檔清理**：支援清空全部雲歸檔連結、刪除已提交連結、刪除單條歸檔連結。
* **設定雲同步**：支援將腳本設定上傳到雲端、從雲端拉取、清空雲端設定，便於多裝置同步腳本偏好。

### 傳輸、雲端下載與分享解析

* **分享解析**：支援解析分享連結，在儲存前查看分享內容，未儲存前不佔用自己的網盤空間。
* **分享文件透視**：支援對分享內容進行遞迴掃描、篩選和預覽，適合在儲存前檢查分享內部結構。
* **分享解析歷史**：可記錄已解析的分享連結和密碼，便於後續快速重新開啟。
* **分享管理**：支援設定分享提取次數上限；達到條件後可自動取消分享，使連結失效。
* **秒傳檔案**：上傳檔案時若伺服器已有相同檔案記錄，可直接完成秒傳，減少重複上傳等待時間。
* **上傳保護**：內建上傳離開提醒，並針對上傳中斷、殘留任務和異常檔案進行保護與清理。
* **雲端下載增強**：支援批次離線連結自動去重；內建磁力連結智慧清洗，可提取 Base32 / Hex 雜湊並剔除干擾文字。
* **TXT 磁力連結提取**：支援預覽 TXT 文字，並從文字中識別磁力連結、HTTP、FTP、ed2k、thunder 等下載連結提交雲端下載。
* **磁力連結預覽**：當磁力連結內容存在公開預覽資訊時，支援顯示檔案數、總大小、類型、雜湊和多張預覽圖。

### 媒體播放增強

* **播放引擎增強**：支援 0.5x - 3.0x 倍速、旋轉翻轉、強制比例、自動跳過片頭片尾、連播 / 循環與進度列縮圖預覽。
* **預設開啟方式**：可在腳本播放器與 PotPlayer 之間選擇預設播放方式，並支援預設影片畫質 / 外部播放畫質設定。
* **音樂播放器**：支援音訊檔案線上播放、播放清單、順序播放、隨機播放、單曲循環、彈窗模式、封面顯示與音量控制。
* **字幕系統**：支援雲端同名字幕載入、本機字幕匯入、線上字幕搜尋；支援字幕軸偏移、字幕位置、字號和背景透明度調整。
* **視覺輔助**：支援圖片或影片目前幀以圖搜圖，可選擇 Google Lens、Yandex、SauceNAO、trace.moe 引擎，用於反查封面、演員、番劇或素材來源。
* **媒體模式**：可在設定中啟用媒體模式，使純影片或純圖片目錄預設按名稱 A-Z 排序，提升劇集 / 漫畫瀏覽連續性。
* **PotPlayer 協議修復助手**：內建 PotPlayer 協議檢測與修復輔助，減少瀏覽器無法喚起外部播放器的問題。

### 下載與分發

* **下載器類型**：支援 Aria2 相容 RPC、Gopeed API、ABDM API 與 IDM 清單匯出，適配不同本機下載工具。

## 常見問題解答

**Q：為什麼安裝了腳本卻沒有顯示藍色懸浮球？**  
**A：** 請按以下順序檢查：

1. 確保腳本管理器中 **PikPak 增強大師** 處於啟用狀態。
2. 確保已登入 PikPak 網頁端帳號。
3. 若使用的是 **Tampermonkey**，請確認瀏覽器擴充功能詳細頁中的 **「允許使用者腳本」** 已勾選。
4. 建議使用 **Edge / Chrome** 最新版瀏覽器。
5. 若以上步驟完成後仍然沒有顯示懸浮球，請暫時關閉其他全部瀏覽器外掛，然後重新整理網頁（F5）再試。

**Q：多層資料夾返回上一層比較麻煩，只能從主介面重新進入嗎？**  
**A：** 不需要。腳本中所有帶路徑列的頁面，均支援**滑鼠滾輪橫向滾動**。

當目錄層級較深，前面的上層資料夾路徑被隱藏時，可以將滑鼠移動到頂部路徑列區域，然後滾動滑鼠滾輪。路徑列會橫向滾動，把被隱藏的上層路徑顯示出來。

顯示後，直接點擊對應的上層資料夾名稱，即可快速返回上一層，或跳轉到任意上層目錄。

**Q：檔案查重 / 資料夾查重時，勾選出來的檔案會自動保留一個原檔案嗎？還是需要我在列表裡再篩選一次？**  
**A：** 檔案查重和資料夾查重裡的勾選項，本質上都是**實際操作項**。查重模式本身不會直接刪除檔案，只會列出可能重複的項目。您需要確認並勾選後，再點擊刪除，才會真正清理。

在查重結果中：

* 每個分組表頭的勾選框，表示選中這一整組；
* 組內排序只負責按時間、大小、路徑、名稱等規則重新排列顯示，不會自動決定刪除誰、保留誰；
* 只有點擊**一鍵勾選**時，腳本才會按規則自動處理：每組保留 1 個，其餘項目會被勾選。

另外，檔案查重裡的**按資料夾選擇**，是將重複列表中某個資料夾內的檔案勾選；**反選**則是將重複列表中除該資料夾以外的檔案勾選。

需要注意的是，檔案查重中的**精準比對**採用雜湊值比對，通常較適合配合一鍵勾選進行清理；而檔案查重中的**時長相似 / 名稱相似**，以及資料夾查重，都屬於相似演算法比對，建議刪除前人工確認，避免誤刪。

**Q：為什麼腳本「精準比對」提示沒有重複檔案，但客戶端裡能看到重複檔案？重新整理網頁或更換瀏覽器也掃描不出來，怎麼辦？**  
**A：** 腳本裡的「精準比對」是按**檔案內容雜湊**判斷的，不是按檔名判斷。只有當檔案雜湊值和檔案大小都一致時，才會被判定為精準重複。

客戶端顯示的「重複檔案」並不一定等同於內容完全相同的檔案。它可能包含同名檔案、檔名後綴帶有 `(1)` / `(2)` 的檔案，或者大小、時長比較接近的檔案。但只要這些檔案的實際內容存在差異，腳本的「精準比對」就不會把它們歸為重複。

如果您想查找這類「看起來重複」的檔案，可以使用檔案查重或資料夾查重中的**名稱相似**模式。

**Q：為什麼有些影片第一次開啟或切換畫質時會提示無法播放，但稍後再開啟又可以播放？**  
**A：** 這類情況通常與 PikPak 官方影片串流的準備狀態或瀏覽器解碼狀態有關，並不一定是腳本本身判斷錯誤。

部分影片在官方網頁端也可能出現第一次打不開、切換畫質短時間失敗、稍後重新開啟又能播放的情況。腳本收到的底層錯誤一般類似：

```text
[VideoError] Code: 4, Msg: PipelineStatus::DEMUXER_ERROR_COULD_NOT_OPEN: FFmpegDemuxer: open context failed
```

這個錯誤無法穩定區分是「官方資源還在準備中」，還是「目前畫質真的不可播放」。如果腳本強行把它當作「正在準備中」並持續重試，可能會讓真正無法播放的影片也被誤判為可播放，從而造成更差的體驗。

因此，腳本目前會保留「目前畫質不可用」的提示，避免誤導使用者。

遇到這種情況時，可以嘗試：

* 等待幾秒後重新開啟影片；
* 切換到其他畫質；
* 使用外部播放嘗試播放。

如果後續能找到官方更可靠的狀態欄位，再繼續優化這類誤報問題。

**Q：批次解壓縮時提示「系統繁忙 / 等待重試」，是腳本出 Bug 了嗎？**  
**A：** 不是。這通常是因為 PikPak 雲端伺服器觸發了並發頻率限制，屬於官方介面目前繁忙，而不是腳本本身異常。

如果某個檔案多次重試後仍然失敗，通常說明該資源在雲端可能已經損壞、受限，或目前伺服器狀態不佳。建議稍後重試，或將檔案下載到本機後再處理。

**Q：批次刪除後再還原，檔案數量正常，但檔案沒有回到原來的資料夾，目錄結構像是亂了，可以一鍵恢復嗎？**  
**A：** 這種情況通常不是操作錯誤，也不是腳本邏輯遺失檔案，更接近 PikPak 官方回收站在**虛擬路徑 / 聚合結果**下執行還原時的表現。

當檔案是在檔案查重、文件透視、資料夾透視、最近新增、收藏夾等**聚合視圖或虛擬路徑**中被批次刪除時，這些檔案本身可能來自多個不同的原始資料夾。刪除後再從回收站還原，官方還原機制有時不會逐個恢復到原來的父目錄，而可能會把大量檔案集中還原到某一個統一目錄中，導致原來的多層級目錄結構被破壞。

建議在檔案查重、文件透視、資料夾透視、最近新增、收藏夾等聚合結果中批次刪除前，先確認這些檔案是否確實不再需要。尤其是檔案來自多個不同目錄時，盡量避免刪除後再依賴回收站批次還原。

**Q：為什麼批次刪除檔案時，部分檔案提示受保護或無法被刪除？**  
**A：** 請先檢查這些檔案是否已被記錄在**資源管理器**中。若您在設定裡開啟了「刪除時跳過管理器中記錄資源」，腳本會將命中的檔案視為受保護項目，以避免誤刪。

**解決方法：**

1. 前往設定中取消勾選該保護規則後，再重新執行刪除；
2. 或點擊工具列 / 側邊欄中的 **「資源管理器」** 入口，選擇 **「立即執行清理」**，對這些記錄項執行強制物理清理。

**Q：為什麼我執行「貼上」操作後，檔案沒有在列表中顯示？**  
**A：** 請先檢查您的網盤剩餘空間是否足夠。PikPak 官方目前對「空間不足」通常採用**靜默攔截**策略，也就是在容量超限時，不一定會彈出明確提示，而是直接在後台中止操作。

因此，如果執行貼上後檔案沒有出現，最常見的原因就是觸發了容量限制。建議先清理空間，再重新嘗試。

**Q：什麼是「多帳號數據遷移」？具體該如何操作？**  
**A：** 該功能可將目前帳號中的檔案或資料夾，快速轉存到您的另一個 PikPak 帳號中。

**操作步驟如下：**

1. 在目前帳號中選中需要遷移的檔案或資料夾；
2. 點擊底部的 **「數據遷移」** 按鈕；
3. 腳本會自動將資料加密打包，並自動登出目前帳號；
4. 隨後，您只需正常登入目標帳號；
5. 登入成功後，腳本會自動偵測本機快取中的遷移包，並彈窗提示您一鍵接收。

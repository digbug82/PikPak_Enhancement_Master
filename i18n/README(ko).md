# 📦 PikPak 강화 마스터

[![Version](https://img.shields.io/badge/Version-1.3.2-0067C0?style=flat-square)](https://github.com/digbug82/PikPak_Enhancement_Master/blob/main/PikPak_Enhancement_Master.user.js) [![License](https://img.shields.io/badge/License-CC--BY--NC--SA--4.0-red?style=flat-square)](https://spdx.org/licenses/CC-BY-NC-SA-4.0.html) [![Platform](https://img.shields.io/badge/Platform-PikPak%20Web-orange?style=flat-square)](https://mypikpak.com/drive/all)

> PikPak 클라우드의 핵심 문제점을 해결하기 위해 설계되었으며, 네이티브 웹 클라이언트를 깊이 있게 제어하고 대체하여 사용자의 상호 작용 경험을 완전히 재구성합니다. 데스크톱 수준의 파일 관리자 표준을 바탕으로 스마트 중복 제거, 일괄 압축 해제, 다중 모드 이름 변경, 다중 계정 원활한 마이그레이션, Aria2 구조화된 푸시 및 몰입형 미디어 재생 엔진을 완벽하게 통합하여 클라우드 데이터 관리를 전례 없이 효율적이고 매끄럽게 만듭니다.

## ✨ 주요 기능 (Features)

### ✨ 경험 및 탐색 엔진

* **인터랙티브 재구성**: 공식 기능을 기반으로 인터페이스를 **Windows 파일 탐색기**와 유사하게 재구성했습니다.
* **터보 모드**: 활성화 시 네이티브 로직을 대신하여, 대량의 파일 환경에서 발생하는 렉과 크래시 문제를 완전히 해결합니다.
* **고급 경로 표시줄**: 마우스 휠 스크롤 및 드롭다운 메뉴를 통한 동일 레벨 전환 및 위치 지정을 지원합니다. 전체 검색 및 분석 제품군이 경로 표시줄에 통합되어 있으며, 경로 표시 및 소스 추적 이동을 지원합니다.
* **경험 향상**: 별표 표시 등 다차원 정렬과 원클릭 **흐림 커버** 및 다크 모드 전환을 지원합니다. 백그라운드에서는 **SWR 전략**을 사용하여 조용하고 매끄럽게 뷰를 새로 고칩니다.
* **백그라운드 인덱싱 및 보호**: 홈페이지에서 파란 점이 깜빡이면 디렉토리 트리가 동기화 중임을 나타냅니다. 시스템 자체적으로 동시 작업 물리 잠금 기능이 있어 충돌하는 작업을 가로채고 불량 데이터 생성을 엄격히 방지합니다. (참고: 기본 폴더인 My Pack은 공식적으로 보호되며, 실수로 삭제하는 등의 작업이 엄격히 방지됩니다.)

### 🌐 전송 및 공유 센터

* **공유 관리**: 추출 횟수 상한 설정을 지원하며, 목표 횟수에 도달하면 링크가 자동으로 무효화되고 공유가 취소됩니다.
* **초고속 업로드**: 로컬 파일/폴더를 웹페이지로 드래그하여 직접 업로드하는 것을 지원하여 공식적인 대용량 파일 제한을 돌파하고 **소형 파일 전송 중단률을 대폭 낮춥니다**.
* **클라우드 다운로드 향상**: 일괄 오프라인 링크 **자동 중복 제거**. 내장된 **마그넷 링크 스마트 정제 엔진**(방해 요소를 제거하기 위해 Base32/Hex 해시 자동 추출). **.torrent** 시드 파일 구문 분석 지원. 제한된 링크에 대해 **웹페이지 스냅샷 저장** 대비책 제공.

### 🎬 몰입형 미디어 향상

* **재생 엔진**: 0.5배속-3.0배속, 회전/반전, 강제 비율, 오프닝/엔딩 자동 건너뛰기 및 **연속/반복** 재생 모드를 지원하며 진행률 표시줄에서 썸네일 미리보기를 지원합니다. 내장된 **워치독(Watchdog)**은 검은 화면이나 지원되지 않는 인코딩이 발생하면 호환되는 화질로 자동 폴백합니다.
* **자막 시스템**: 클라우드의 동일한 이름의 자막, 로컬 파일 및 크로스 사이트 온라인 검색 로드를 지원합니다. 자막 싱크의 밀리초 단위 오프셋 미세 조정을 지원하며 로컬 텍스트 **드래그 앤 드롭 구문 분석** 마운트를 지원합니다.
* **시각 보조**: 내장된 다중 엔진은 이미지 또는 비디오의 현재 프레임에 대한 **이미지로 검색(Reverse Image Search)**을 지원합니다. 설정에서 "미디어 모드"를 활성화하면 시리즈/만화 폴더가 이름(A-Z) 순서대로 자동 정렬됩니다.

### ⚡ 다운로드 및 배포

* **외부 직접 연결**: 원클릭으로 비디오 스트림 직접 링크를 가져오거나 PotPlayer를 호출하여 재생하는 것을 지원합니다. RPC 프로토콜을 통해 파일을 **Aria2/Motrix** 노드로 원클릭 푸시하는 것을 지원합니다.
* **배포 향상**: 폴더를 Aria2/Motrix로 푸시할 때 **클라우드 드라이브 트리 디렉토리 구조를 자동 복원**합니다. 장기 연결 모니터링을 지원하며 오류 발생 시 자동으로 오류 목록을 내보냅니다. **폴더 다운로드 필터링** 설정을 지원합니다.

## ❓ 자주 묻는 질문 (FAQ)

**Q: 스크립트를 설치했는데 파란색 플로팅 볼이 표시되지 않는 이유는 무엇인가요?**  
**A:** PikPak 웹 클라이언트에 로그인했는지 확인하세요. 로그인했는데도 표시되지 않으면 웹페이지(F5)를 새로 고침하거나 AdGuard / uBlock Origin과 같은 광고 차단 확장 프로그램에 의해 차단되었는지 확인하세요. 차단 프로그램을 일시적으로 비활성화하거나 이 사이트를 화이트리스트에 추가해 보세요.

**Q: "터보 모드(Turbo Mode)"란 무엇이며 활성화가 권장되는 이유는 무엇인가요?**  
**A:** 활성화하면 스크립트가 웹페이지를 전체 화면으로 덮고 모든 기본 웹 기능을 완전히 제어합니다. 이 모드는 원본 웹페이지의 메모리 소모가 심한 동기화 로직을 물리적으로 차단하여 대량의 파일이 있을 때 발생하는 렉과 크래시 문제를 완전히 해결합니다. **참고:** 터보 모드에서는 최소화 및 닫기 버튼이 자동으로 숨겨집니다(원래 웹으로 돌아가려면 설정에서 체크 해제해야 함). **권장:** 파일이 많은 경우 극강의 응답 속도와 부드러운 경험을 얻기 위해 이 모드를 활성화하는 것을 강력히 권장합니다.

**Q: 홈 페이지의 My Pack에 "기본" 태그가 표시되고 별표 표시, 복사, 이동, 이름 변경 또는 삭제를 할 수 없는 이유는 무엇입니까?**  
**A:** 공식적인 보안 보호 메커니즘입니다. 스크립트는 이를 바탕으로 기본 폴더 My Pack의 공유 제한을 해제했습니다.

**Q: 스크립트의 "파일 중복 제거"와 "폴더 중복 제거"의 근본적인 차이점은 무엇입니까? 실제 사용 시 어떻게 선택해야 하나요?**  
**A:** **파일 중복 제거:** "단일 파일" 수준을 대상으로 합니다. 해시 정밀 일치, 비디오 재생 시간 유사도, 이름 유사도 3가지 모드 알고리즘을 사용하여 클라우드 드라이브에 흩어져 있는 중복 파일(예: 중복 저장된 단일 에피소드 비디오, 단일 이미지 등)을 정리하는 데 적합합니다.<br>
**폴더 중복 제거:** "폴더 전체 구조" 수준을 대상으로 합니다. 이름 일치, 내부 파일 중복도(유사도 일치), 하위 세트 중복(포함률 일치) 3가지 모드 알고리즘을 사용합니다. 폴더의 "내부 본질"을 파악할 수 있어 중복 저장된 "전체 시리즈", "전체 이미지 팩" 또는 "다중 레벨 데이터"를 정리하는 데 매우 적합합니다. 두 폴더의 이름이 완전히 다르더라도 내부 파일이 매우 일치하거나 포함 관계가 있으면 정확하게 잡아낼 수 있습니다.

> **권장:** 파일 중복 제거의 정밀 중복 제거는 해시 일치이므로 원클릭으로 선택하여 삭제할 수 있습니다. 파일 중복 제거의 재생 시간 및 이름 일치, 폴더 중복 제거의 3가지 모드는 모두 유사도 일치이므로 실수로 삭제되지 않도록 주의해서 원클릭 선택을 사용하십시오.

**Q: 파일 중복 제거 결과로 나오는 중복 파일이 점점 많아지는 이유는 무엇인가요? 전체 인덱스의 패킷이 손실되었나요?**  
**A:** 안심하세요. 전체 인덱스 패킷 손실이나 데이터 오류가 아닙니다. 중복 제거 결과 중 "정밀 일치" 부분의 수는 항상 안정적입니다. 중복 제거 결과가 증가하는 이유는 비디오 재생 시간에 기반한 "유사도 일치"가 더 포괄적으로 변했기 때문입니다. PikPak의 일부 원본 데이터 인터페이스는 비디오 재생 시간을 직접 제공하지 않아 초기에는 데이터 누락으로 인해 많은 중복 비디오를 건너뛰었습니다. 이를 위해 스크립트에는 비디오 메타데이터 읽기를 시뮬레이션하여 정확한 재생 시간을 추출하고 브라우저 로컬에 영구적으로 저장하는 백그라운드 비동기 스니핑 알고리즘이 내장되어 있습니다. 사용 시간이 늘어남에 따라 로컬 재생 시간 데이터베이스가 점점 완성되어 중복 제거 엔진이 보완된 데이터를 기반으로 이름은 다르지만 내용 길이가 동일한 숨겨진 중복 리소스를 더 많이 찾아낼 수 있게 되어 결과가 점점 많아지고 정확해지는 것입니다.

**Q: "이미지로 검색" 시 이미지를 직접 업로드할 수 없고 붙여넣기 하라는 메시지가 표시되는 경우가 있는 이유는 무엇입니까?**  
**A:** 이는 브라우저의 보안 정책(CORS 교차 도메인 제한) 때문입니다. 스크립트가 클라우드 드라이브 이미지를 이미지 검색 서버로 직접 푸시할 수 없는 경우 자동으로 현재 화면 스크린샷을 클립보드에 복사합니다. 팝업된 검색 페이지에서 `[Ctrl+V]`를 누르기만 하면 인식이 완료됩니다.

**Q: 스크립트를 사용하여 비디오를 재생할 때 소리만 나고 화면이 나오지 않는 이유는 무엇입니까?**  
**A:** 이는 일반적으로 해당 비디오가 m3u8 다중 오디오/다중 트랙 스트리밍 미디어에 속하기 때문입니다. 클라우드 트랜스코딩 후 사운드와 화면 트랙이 분리되었으며, 웹 플레이어는 현재 이러한 특수 비디오 스트림의 온라인 다중 트랙 구문 분석을 지원하지 않습니다. 오른쪽 하단의 "외부 재생" 버튼을 클릭하여 PotPlayer와 같은 로컬 전문 플레이어를 직접 호출하여 스트리밍 재생하는 것을 권장합니다.

**Q: 해상도 메뉴에서 "원본(고속)"과 일반 "원본"의 차이점은 무엇입니까?**  
**A:** 두 화질은 거의 완전히 동일하며 가장 큰 차이점은 기본 디코딩 및 전송 프로토콜에 있습니다. "원본(고속)"은 공식적으로 스트리밍 미디어에 최적화된 링크를 사용합니다. 일반적으로 PotPlayer와 같은 외부 플레이어를 호출할 때 "고속"이라는 단어가 있는 링크를 사용하면 가장 빠른 로딩 응답과 가장 부드러운 드래그 경험을 얻을 수 있습니다. 웹 재생이든 외부 스트리밍이든 이 옵션을 우선적으로 선택하는 것이 좋습니다.

**Q: 모바일 앱이나 공식 웹에서 방금 수행한 작업(예: 업로드, 삭제, 이동)이 스크립트의 중복 제거 또는 분석 목록에 실시간으로 동기화되지 않는 이유는 무엇입니까?**  
**A:** 스크립트는 극강의 검색 및 분석 속도를 보장하기 위해 메모리 스냅샷 메커니즘을 채택했으며 매번 수만 개의 파일 정보를 다시 가져오지 않습니다. 해결 방법: 다른 클라이언트에서 너무 많은 파일을 수정한 경우 웹 페이지를 새로 고쳐 전체 인덱스를 다시 가져오십시오. 스크립트 작업 중에는 여러 기기에서 동시에 데이터를 수정하지 마십시오.

**Q: 일괄 압축 해제 시 한 번에 여러 파일에 다른 비밀번호를 설정할 수 있나요?**  
**A:** 가능합니다. 스마트 일치 로직: 알려진 비밀번호를 "비밀번호 금고"에 미리 저장해 두기만 하면 스크립트는 일괄 압축 해제 시 각 압축 파일에 대해 금고의 모든 비밀번호를 자동으로 시도합니다. 일치하는 경우 수동 개입 없이 자동으로 시작됩니다.

**Q: 일괄 압축 해제 시 "시스템이 사용 중입니다/재시도 대기 중"이라는 메시지가 표시되면 스크립트 버그입니까?**  
**A:** 아닙니다. 이는 보통 PikPak 클라우드 서버가 동시 실행 빈도 제한을 트리거했기 때문입니다. 이 프롬프트는 클라우드 드라이브 공식 인터페이스도 현재 사용 중임을 의미합니다. 파일 재시도를 여러 번 해도 계속 실패한다면 클라우드의 해당 리소스가 손상되었거나 제한되었을 수 있으므로 나중에 다시 시도하거나 다시 업로드/로컬로 다운로드하여 처리하는 것을 권장합니다.

**Q: 내보낸 구성 JSON에는 어떤 정보가 포함되어 있나요?**  
**A:** JSON 파일에는 중복 제거 환경설정, 블랙/화이트리스트 규칙, 비밀번호 금고 등 강화 마스터에서 설정한 로컬 구성 데이터만 포함되며 클라우드 드라이브 계정 비밀번호는 **포함되지 않으므로** 기기 간 마이그레이션에 안심하고 사용할 수 있습니다.

**Q: 백업을 가져오면 기존 구성이나 목록을 덮어쓰나요?**  
**A:** 덮어쓰지 않습니다. 스크립트가 스마트 병합 로직으로 업그레이드되어 가져올 때 리소스 관리자, 기록 등의 목록 항목을 로컬 데이터와 자동으로 중복 제거 및 병합하며, 기본 설정 항목(검색 엔진, Aria2 주소 등)만 가져온 파일을 기준으로 하므로 로컬에 축적된 데이터가 손실되지 않습니다.

**Q: 일괄 삭제 시 일부 파일이 보호되거나 삭제할 수 없다는 메시지가 표시되는 이유는 무엇입니까?**  
**A:** 이러한 파일이 리소스 관리자에 기록되어 있는지 확인하십시오. 설정에서 "삭제 시 관리자에 기록된 리소스 건너뛰기"를 활성화한 경우 스크립트는 일치하는 파일을 보호 항목으로 간주하여 오작동을 방지합니다.<br>
해결 방법: 
1. 설정에서 해당 보호 규칙을 선택 취소한 후 삭제합니다.
2. 또는 도구 모음/사이드바의 "리소스 관리자" 항목을 클릭하고 "지금 정리 실행"을 클릭하여 이러한 기록 항목을 물리적으로 강제 삭제합니다.

**Q: "붙여넣기" 작업을 수행한 후 파일이 목록에 표시되지 않는 이유는 무엇입니까?**  
**A:** 클라우드 드라이브의 남은 여유 공간이 해당 파일을 수용하기에 충분한지 확인하십시오. PikPak 공식은 현재 "공간 부족" 상황에 대해 **자동 차단(Silent Interception)** 전략을 취하고 있습니다. 즉, 한도를 초과할 때 구체적인 오류 프롬프트를 띄우지 않고 백그라운드에서 직접 작업을 중단합니다. 붙여넣기 후 파일이 나타나지 않으면 일반적으로 용량 제한이 트리거되었기 때문이며 공간을 확보한 후 다시 시도하는 것이 좋습니다.

**Q: 클라우드 다운로드 팝업의 "차단 방지 마그넷 링크 자동 복구" 기능은 무엇이며 언제 켜는 것이 좋습니까?**  
**A:** 이 기능은 링크에 섞여 있는 한자, 이모티콘, 기호 등 문자 간섭을 자동으로 제거하고 특성 코드를 스마트하게 식별 및 추출하여 표준 형식으로 복원할 수 있습니다.<br>
**권장: 이 기능은 기본적으로 선택되어 유지하는 것이 좋습니다. 소셜 플랫폼에서 얻은, 관련 없는 텍스트가 많이 섞여 있거나 "헤더가 없는" 오염된 마그넷 링크를 자동으로 처리할 수 있으며, `magnet:?`로 시작하는 표준 마그넷 링크가 이미 포함된 경우 복구를 건너뛰고 정상적으로 다운로드합니다.**

**Q: Aria2/Motrix 또는 브라우저를 사용하여 클라우드 드라이브 폴더를 다운로드할 때 계층 구조가 유지됩니까?**  
**A:** **Aria2/Motrix는 계층을 유지합니다**. 스크립트는 디렉토리 계층 복제 기술을 채택하여 다운로드가 완료되면 로컬 하드 드라이브가 클라우드 드라이브의 트리 구조를 완벽하게 복원하며, 클라우드 드라이브 폴더 이름에서 Windows에서 지원하지 않는 특수 문자(예: `:`, `*`, `?` 등)를 자동으로 정리하고 다운로드 오류가 발생하지 않도록 중복 경로를 스마트하게 줄입니다.  
**브라우저 다운로드는 계층을 유지하지 않습니다**. 브라우저의 기본 다운로드 프로토콜의 한계로 인해 폴더 내의 모든 파일은 동일한 디렉토리에 "평면"으로 다운로드됩니다. **권장:** 복잡한 폴더 구조를 유지해야 하는 경우 반드시 Aria2 푸시를 우선적으로 선택하십시오.

**Q: "다중 계정 데이터 마이그레이션"이란 무엇입니까? 구체적으로 어떻게 조작해야 합니까?**  
**A:** 이 기능을 사용하면 현재 계정의 파일을 다른 계정으로 원활하고 빠르게 전송할 수 있습니다.
1. 현재 계정에서 전송할 파일이나 폴더를 선택하고 하단의 "데이터 마이그레이션" 버튼을 클릭합니다.
2. 스크립트가 자동으로 데이터를 암호화하여 패키징하고 현재 계정에서 자동으로 로그아웃합니다.
3. 그런 다음 [대상 계정]으로 정상적으로 로그인하기만 하면 로그인 성공 후 스크립트가 로컬에 캐시된 마이그레이션 패키지를 자동 감지하고 원클릭으로 수신하라는 팝업 프롬프트를 표시합니다.

* **다중 계정 데이터 마이그레이션** 기능 추가: 대량의 파일을 암호화 패키징하여 다른 계정으로 원활하게 전송할 수 있는 기능을 지원합니다.
* **영구 삭제** 메커니즘 도입: 삭제 시 휴지통을 거치지 않고 즉시 클라우드 공간을 확보할 수 있습니다.
* 이미지 확대/축소 또는 긴 이미지 모드에서 드래그 시 가장자리 제한 계산 오류가 발생하던 문제를 수정했습니다.
* '최근 추가', '분석' 등 특수 뷰에서 경로 표시줄의 폴더 목록 정렬이 헤더 설정과 일치하지 않거나 정렬 모드가 작동하지 않던 문제를 전면 수정했습니다.
* 다양한 해상도의 화면에 따른 반응형 레이아웃 전환 임계값과 상호작용 디테일을 최적화했습니다.
* 특정 상황에서 모달 창의 레이어가 서로 겹치거나 가려지던 문제를 수정했습니다.
* 미로그인 상태 감지 로직을 강화하여 네트워크 재시도 무한 루프 현상을 방지했습니다.

* 경로 검색 시 하이라이트가 작동하지 않던 버그 및 설정에서 "초고속 모드" 변경 시 다른 설정 항목이 차단되던 문제를 수정했습니다.

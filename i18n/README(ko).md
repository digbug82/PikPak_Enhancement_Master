## 주요 기능

### 경험 및 내비게이션 강화

* **사이드 버튼 탐색**: 마우스 사이드 버튼의 앞으로 / 뒤로 가기를 지원하여 서로 다른 폴더 계층 사이를 빠르게 전환할 수 있습니다.
* **고급 경로 표시줄**: 휠 스크롤, 같은 계층 폴더 드롭다운 전환, 경로 표시 및 원본 경로 추적 이동을 지원합니다.
* **전체 드라이브 검색 강화**: 파일 이름과 파일 경로로 검색할 수 있고, 키워드 일부를 빨간색으로 강조하며, 긴 파일 이름은 적중 키워드 주변 내용을 우선 표시합니다.
* **탐색 환경 설정**: 정렬 환경 설정, 보기 환경 설정, 탐색 위치 유지, 미디어 모드, 버튼 텍스트 숨기기, 미디어 커버 썸네일 흐림, 야간 모드를 지원합니다.

* **일괄 이름 변경**: 정규식 치환 / 삭제, 에피소드 일련번호, 텍스트 형식화, FC2 규칙 이름 지정, 광고 접두어 제거, MIME 기반 확장자 스마트 복구를 지원합니다.

* **필터 강화**: 크기, 유형, 경로, 키워드 등 조건으로 필터링할 수 있으며, 한 번에 선택, 반전 선택, 폴더별 선택, 그룹 내 정렬 등 일괄 보조 기능을 제공합니다.

* **리소스 관리자**: 사용자 지정 리소스 목록을 지원합니다. 블랙리스트로 정리를 실행하거나, 일괄 삭제 시 보호 목록으로 사용해 적중 항목을 자동으로 건너뛸 수 있습니다.

### 링크 북마크 및 클라우드 아카이브

* **링크 북마크**: 링크 북마크는 북마크 링크, 스크립트 동기화 데이터, 클라우드 아카이브 마그넷 링크, 클라우드 설정을 함께 저장합니다.
* **클라우드 아카이브**: 선택한 파일의 추적 가능한 마그넷 링크를 “링크 북마크 > 클라우드 아카이브”에 보관할 수 있으며, 원본 파일을 삭제하여 공간을 절약할 수 있습니다.
* **클라우드 아카이브 복원 파싱**: 보관된 마그넷 링크를 다시 클라우드 다운로드 작업으로 제출하고 제출 상태를 기록할 수 있습니다.
* **클라우드 아카이브 정리**: 모든 클라우드 아카이브 링크 비우기, 제출된 링크 삭제, 개별 아카이브 링크 삭제를 지원합니다.
* **설정 클라우드 동기화**: 스크립트 설정을 클라우드에 업로드하고, 클라우드에서 가져오며, 클라우드 설정을 비울 수 있어 여러 기기 간 스크립트 환경 설정 동기화에 편리합니다.

### 전송, 클라우드 다운로드 및 공유 분석

* **공유 분석**: 공유 링크를 분석하고 저장 전에 공유 내용을 볼 수 있으며, 저장 전에는 내 클라우드 드라이브 공간을 사용하지 않습니다.
* **공유 파일 투시**: 공유 내용에 대한 재귀 스캔, 필터링, 미리보기를 지원하여 저장 전에 공유 내부 구조를 확인하기 좋습니다.
* **공유 분석 기록**: 이미 분석한 공유 링크와 비밀번호를 기록하여 나중에 빠르게 다시 열 수 있습니다.
* **공유 관리**: 공유 접근 횟수 상한을 설정할 수 있으며, 조건에 도달하면 공유를 자동으로 취소해 링크를 무효화할 수 있습니다.
* **즉시 업로드**: 서버에 동일한 파일 기록이 있으면 업로드를 즉시 완료할 수 있어 반복 업로드 대기 시간을 줄일 수 있습니다.
* **업로드 보호**: 업로드 중 페이지 이탈 알림을 내장하고, 업로드 중단, 잔여 작업, 비정상 파일을 보호하고 정리합니다.
* **클라우드 다운로드 강화**: 일괄 오프라인 링크 자동 중복 제거를 지원하며, 내장된 스마트 마그넷 정리는 Base32 / Hex 해시를 추출하고 방해 텍스트를 제거할 수 있습니다.
* **TXT 마그넷 추출**: TXT 텍스트 미리보기를 지원하고, 텍스트에서 마그넷, HTTP, FTP, ed2k, thunder 등의 다운로드 링크를 인식해 클라우드 다운로드로 제출합니다.
* **마그넷 미리보기**: 마그넷 내용에 공개 미리보기 정보가 있을 때 파일 수, 전체 크기, 유형, 해시, 여러 미리보기 이미지를 표시합니다.

### 미디어 재생 강화

* **재생 엔진 강화**: 0.5x - 3.0x 배속, 회전 / 뒤집기, 강제 비율, 오프닝 / 엔딩 자동 건너뛰기, 연속 재생 / 반복, 진행률 표시줄 썸네일 미리보기를 지원합니다.
* **기본 열기 방식**: 스크립트 플레이어와 PotPlayer 중 기본 재생 방식을 선택할 수 있으며, 기본 비디오 화질 / 외부 재생 화질 설정을 지원합니다.
* **음악 플레이어**: 오디오 파일 온라인 재생, 재생 목록, 순차 재생, 랜덤 재생, 한 곡 반복, 팝업 모드, 커버 표시 및 볼륨 제어를 지원합니다.
* **오디오 팝업 모드**: 음악 플레이어를 작은 창으로 분리하면서 재생 제어, 커버 표시, 테마 적용을 유지할 수 있습니다.
* **자막 시스템**: 클라우드 동일 이름 자막 로드, 로컬 자막 가져오기, 온라인 자막 검색을 지원하며, 자막 싱크 오프셋, 위치, 글꼴 크기, 배경 투명도 조정을 지원합니다.
* **시각 보조**: 이미지 또는 현재 비디오 프레임으로 이미지 검색을 수행할 수 있으며, Google Lens, Yandex, SauceNAO, trace.moe 엔진을 선택해 커버, 배우, 애니메이션 또는 소재 출처를 역검색할 수 있습니다.
* **미디어 모드**: 설정에서 미디어 모드를 활성화하면 비디오만 있거나 이미지만 있는 디렉터리를 기본으로 이름 A-Z 정렬하여 에피소드 / 만화 탐색 연속성을 높일 수 있습니다.
* **PotPlayer 프로토콜 복구 도우미**: 내장된 PotPlayer 프로토콜 감지 및 복구 보조 기능으로 브라우저가 외부 플레이어를 호출하지 못하는 문제를 줄입니다.

### 다운로드 및 배포

* **다운로더 유형**: Aria2 호환 RPC, Gopeed API, ABDM API 및 IDM 목록 내보내기를 지원하여 다양한 로컬 다운로드 도구에 맞출 수 있습니다.
* **디렉터리 구조 복원**: 전체 폴더를 보낼 때 클라우드 드라이브의 트리형 디렉터리 구조를 자동으로 복원하여 다운로드 후 폴더가 납작해지는 것을 방지합니다.
* **비디오 다운로드 가속**: 비디오 다운로드는 재생 경로를 우선 사용해 원본 화질 직접 링크를 가져오고 다운로드 속도를 높일 수 있습니다.
* **다운로드 필터**: 파일 크기, 확장자, 이름 키워드로 파일을 필터링하며, 브라우저 다운로드와 다운로더 배포에 적용됩니다.
* **다운로드 직접 링크 가속 도메인**: 다운로드 가속 도메인을 지원하며, 원본 링크 이어붙이기와 URL 매개변수 전달 두 가지 리버스 프록시 재작성 모드를 제공합니다.
* **예외 처리**: 전송 시 0KB 무효 파일을 자동으로 건너뛰고 실패 목록 내보내기를 지원합니다.

## 설치 가이드

## 자주 묻는 질문

**Q: 스크립트를 설치했는데 파란색 플로팅 공이 보이지 않는 이유는 무엇인가요?**  
**A:** 다음 순서대로 확인해 주세요.

1. 유저스크립트 관리자에서 **PikPak Enhancement Master**가 활성화되어 있는지 확인합니다.
2. PikPak 웹 계정에 로그인되어 있는지 확인합니다.
3. **Tampermonkey**를 사용하는 경우, 브라우저 확장 프로그램 상세 페이지에서 **"사용자 스크립트 허용"**이 체크되어 있는지 확인합니다.
4. **Edge / Chrome** 최신 버전 브라우저 사용을 권장합니다.
5. 위 단계를 완료해도 플로팅 공이 표시되지 않으면, 다른 모든 브라우저 확장 프로그램을 임시로 끈 뒤 페이지를 새로고침(F5)하고 다시 시도하세요.

**Q: 여러 단계의 폴더에서 상위 단계로 돌아가기 번거롭습니다. 메인 화면에서 다시 들어가야 하나요?**  
**A:** 필요 없습니다. 스크립트의 경로 표시줄이 있는 모든 페이지는 **마우스 휠 가로 스크롤**을 지원합니다.

디렉터리 계층이 깊어 앞쪽 상위 폴더 경로가 숨겨진 경우, 마우스를 상단 경로 표시줄 영역으로 이동한 뒤 마우스 휠을 굴리면 됩니다. 경로 표시줄이 가로로 스크롤되어 숨겨진 상위 경로가 표시됩니다.

표시된 뒤 해당 상위 폴더 이름을 직접 클릭하면 이전 단계로 빠르게 돌아가거나 임의의 상위 디렉터리로 이동할 수 있습니다.

**Q: 파일 중복 확인 / 폴더 중복 확인에서 선택된 파일은 원본 하나를 자동으로 남기나요, 아니면 목록에서 다시 필터링해야 하나요?**  
**A:** 파일 중복 확인과 폴더 중복 확인의 선택 항목은 본질적으로 **실제 작업 대상**입니다. 중복 확인 모드 자체는 파일을 직접 삭제하지 않고, 중복 가능성이 있는 항목만 나열합니다. 확인 후 선택하고 삭제를 클릭해야 실제로 정리됩니다.

중복 확인 결과에서:

* 각 그룹 헤더의 체크박스는 해당 그룹 전체 선택을 의미합니다;
* 그룹 내 정렬은 시간, 크기, 경로, 이름 등의 규칙으로 표시 순서만 다시 배열하며, 무엇을 삭제하고 무엇을 남길지 자동 결정하지 않습니다;
* **한 번에 선택**을 클릭할 때만 스크립트가 규칙에 따라 각 그룹에서 1개를 남기고 나머지 항목을 선택합니다.

또한 파일 중복 확인의 **폴더별 선택**은 중복 목록에서 특정 폴더 안의 파일을 선택하는 기능이고, **반전 선택**은 해당 폴더 밖의 파일을 선택하는 기능입니다.

주의할 점은 파일 중복 확인의 **정밀 일치**는 해시값 매칭을 사용하므로 보통 한 번에 선택과 함께 정리하기에 적합합니다. 반면 파일 중복 확인의 **재생시간 유사 / 이름 유사** 및 폴더 중복 확인은 유사도 알고리즘 기반이므로 삭제 전 수동 확인을 권장합니다.

**Q: 스크립트의 "정밀 일치"는 중복 파일이 없다고 하는데, 클라이언트에서는 중복 파일이 보입니다. 새로고침하거나 브라우저를 바꿔도 스캔되지 않으면 어떻게 하나요?**  
**A:** 스크립트의 "정밀 일치"는 **파일 내용 해시**로 판단하며, 파일 이름으로 판단하지 않습니다. 파일 해시값과 파일 크기가 모두 같을 때만 정밀 중복으로 판정됩니다.

클라이언트가 표시하는 "중복 파일"이 반드시 내용이 완전히 같은 파일이라는 뜻은 아닙니다. 같은 이름의 파일, 파일명 뒤에 `(1)` / `(2)`가 붙은 파일, 또는 크기와 재생시간이 비교적 가까운 파일이 포함될 수 있습니다. 실제 내용에 차이가 있으면 스크립트의 "정밀 일치"는 이를 중복으로 분류하지 않습니다.

이런 "겉보기 중복" 파일을 찾고 싶다면 파일 중복 확인 또는 폴더 중복 확인의 **이름 유사** 모드를 사용할 수 있습니다.

**Q: 일부 비디오는 처음 열거나 화질을 전환할 때 재생 불가 메시지가 뜨지만, 잠시 뒤 다시 열면 재생되는 이유는 무엇인가요?**  
**A:** 이런 상황은 대개 PikPak 공식 비디오 스트림의 준비 상태나 브라우저 디코딩 상태와 관련이 있으며, 반드시 스크립트의 판단 오류는 아닙니다.

일부 비디오는 공식 웹에서도 처음 열리지 않거나, 화질 전환 직후 잠시 실패했다가 나중에 다시 열면 재생될 수 있습니다. 스크립트가 받는 하위 오류는 보통 다음과 유사합니다.

```text
[VideoError] Code: 4, Msg: PipelineStatus::DEMUXER_ERROR_COULD_NOT_OPEN: FFmpegDemuxer: open context failed
```

이 오류는 "공식 리소스가 아직 준비 중"인지 "현재 화질이 실제로 재생 불가"인지 안정적으로 구분할 수 없습니다. 스크립트가 이를 강제로 "준비 중"으로 보고 계속 재시도하면, 실제로 재생할 수 없는 비디오도 재생 가능으로 오판되어 더 나쁜 경험을 만들 수 있습니다.

따라서 현재 스크립트는 사용자를 오도하지 않기 위해 "현재 화질을 사용할 수 없음" 안내를 유지합니다.

이 상황에서는 다음을 시도할 수 있습니다.

* 몇 초 기다린 뒤 비디오를 다시 엽니다;
* 다른 화질로 전환합니다;
* 외부 재생을 사용해 봅니다.

향후 공식적으로 더 신뢰할 수 있는 상태 필드를 찾으면 이런 오탐 문제를 계속 최적화할 수 있습니다.

**Q: 일괄 압축 해제 중 "시스템 사용 중 / 재시도 대기"가 표시됩니다. 스크립트 버그인가요?**  
**A:** 아닙니다. 보통 PikPak 클라우드 서버가 동시 실행 빈도 제한을 트리거한 것으로, 공식 API가 바쁜 상태이지 스크립트 자체의 예외는 아닙니다.

어떤 파일이 여러 번 재시도해도 실패한다면, 해당 리소스가 클라우드에서 손상되었거나 제한되었거나 현재 서버 상태가 좋지 않을 가능성이 큽니다. 나중에 다시 시도하거나 파일을 로컬로 다운로드한 뒤 처리하는 것을 권장합니다.

**Q: 일괄 삭제 후 복원했더니 파일 수는 정상인데 원래 폴더로 돌아가지 않았고 디렉터리 구조가 뒤섞인 것 같습니다. 한 번에 복구할 수 있나요?**  
**A:** 이 상황은 보통 조작 실수도 아니고 스크립트가 파일을 잃어버린 것도 아닙니다. PikPak 공식 휴지통이 **가상 경로 / 집계 결과**에서 복원을 수행할 때 보이는 동작에 가깝습니다.

파일 중복 확인, 파일 투시, 폴더 투시, 최근 추가, 즐겨찾기 같은 **집계 보기 또는 가상 경로**에서 파일을 일괄 삭제한 경우, 파일들이 원래 여러 다른 폴더에서 왔을 수 있습니다. 삭제 후 휴지통에서 복원하면 공식 복원 메커니즘이 각 항목을 원래 부모 폴더로 하나씩 복원하지 않고, 많은 파일을 하나의 통합 디렉터리로 복원하여 원래 다층 디렉터리 구조가 깨질 수 있습니다.

파일 중복 확인, 파일 투시, 폴더 투시, 최근 추가, 즐겨찾기 등 집계 결과에서 일괄 삭제하기 전에는 해당 파일이 정말 더 이상 필요 없는지 먼저 확인하세요. 특히 파일이 여러 디렉터리에서 온 경우, 삭제 후 휴지통 일괄 복원에 의존하는 것은 피하는 편이 좋습니다.

**Q: 파일을 일괄 삭제할 때 일부 파일이 보호되었거나 삭제할 수 없다고 표시되는 이유는 무엇인가요?**  
**A:** 먼저 이 파일들이 **리소스 관리자**에 기록되어 있는지 확인하세요. 설정에서 "삭제 시 관리자에 기록된 리소스 건너뛰기"를 활성화했다면, 스크립트는 적중 파일을 보호 항목으로 간주해 실수 삭제를 피합니다.

**해결 방법:**

1. 설정으로 이동해 해당 보호 규칙을 해제한 뒤 다시 삭제를 실행합니다;
2. 또는 도구 모음 / 사이드바의 **"리소스 관리자"** 항목을 클릭하고 **"지금 정리 실행"**을 선택해 해당 기록 항목을 강제로 물리 정리합니다.

**Q: "붙여넣기" 작업 후 파일이 목록에 표시되지 않는 이유는 무엇인가요?**  
**A:** 먼저 클라우드 드라이브의 남은 공간이 충분한지 확인하세요. PikPak 공식 서비스는 현재 "공간 부족"에 대해 대개 **무음 차단** 방식을 사용합니다. 즉 용량 초과 시 명확한 알림을 표시하지 않고 백그라운드에서 작업을 중단할 수 있습니다.

따라서 붙여넣기 후 파일이 나타나지 않는 가장 흔한 원인은 용량 제한입니다. 먼저 공간을 정리한 뒤 다시 시도하세요.

**Q: "다중 계정 데이터 마이그레이션"이란 무엇이며 어떻게 사용하나요?**  
**A:** 이 기능은 현재 계정의 파일 또는 폴더를 다른 PikPak 계정으로 빠르게 전송할 수 있게 합니다.

**작업 단계:**

1. 현재 계정에서 마이그레이션할 파일 또는 폴더를 선택합니다;
2. 하단의 **"데이터 마이그레이션"** 버튼을 클릭합니다;
3. 스크립트가 데이터를 자동으로 암호화해 패키징하고 현재 계정에서 로그아웃합니다;
4. 이후 대상 계정으로 정상 로그인합니다;
5. 로그인 성공 후 스크립트가 로컬 캐시에 있는 마이그레이션 패키지를 자동 감지하고, 한 번에 수신할 수 있도록 팝업을 표시합니다.

## 개인정보 및 보안 안내

* **로컬 우선**: 이 스크립트의 모든 핵심 기능은 브라우저를 통해 PikPak 공식 API와 직접 상호작용합니다. 계정 Token, 압축 해제 비밀번호 저장소 및 대부분의 로컬 설정 데이터는 기본적으로 로컬 브라우저 환경에 저장됩니다.
* **수집 없음**: 스크립트는 사용자 개인정보를 **능동적으로 수집하지 않으며**, 파일 정보나 계정 자격 증명을 **절대** 제3자 서버로 업로드하지 않습니다.
* **제3자 인터페이스**: "온라인 자막 검색", "이미지 검색", "마그넷 미리보기" 같은 확장 기능을 사용할 때에만 필요한 검색 키워드, 이미지 특징 매개변수 또는 마그넷 특징 정보를 관련 공공 서비스로 전송하며, 개인 신원 정보는 포함하지 않습니다.
* **멤버십 권한 경계**: 이 스크립트는 PikPak 웹 조작 경험을 강화하기 위한 것이며, **PikPak 멤버십 권한을 크랙, 우회 또는 위조하는 기능이 없습니다**. 또한 공식 서비스 규칙을 위반하는 어떠한 멤버십 해제 기능도 제공하지 않습니다.

* 개선 **오프라인 다운로드**, 오프라인 작업에 경량 탐지와 상태 동기화를 추가했으며, 작업 추가 및 삭제 시 목록을 비우고 전체 새로고침하지 않도록 개선했습니다.

* 개선 **새로고침 및 가상 장면 상태 유지**, 수동 새로고침, 네트워크 복구, 인증 복구, 검색 결과, 최근 항목, 기록, 오프라인, 공유 파싱, 링크 북마크 등 장면의 목록 안정성을 개선하여 위치 튐, 빈 화면, 오작동 새로고침을 줄였습니다.
* 개선 **UI 및 안정성**, CSS와 SVG 아키텍처를 정리했습니다.

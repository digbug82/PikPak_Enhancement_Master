### ✨ 경험 및 탐색 엔진

* **인터랙션 재구성**: Windows 파일 탐색기 스타일의 인터페이스 디자인으로 조작 편의성 극대화

## ❓ 자주 묻는 질문 (FAQ)

**Q: 스크립트를 설치했는데 왜 파란색 부유 구슬이 나타나지 않나요?**
**A:** PikPak 웹 버전에 로그인했는지 확인해 주세요. 로그인 상태임에도 나타나지 않는다면 페이지를 새로고침(F5) 하거나, AdGuard / uBlock Origin 같은 광고 차단 플러그인이 오작동하여 차단 중인지 확인 후 해당 사이트를 허용 목록(화이트리스트)에 추가해 주세요.

**Q: “초고속 모드 (Turbo Mode)”가 무엇이며 왜 권장되나요?**
**A:** 이 모드는 본 스크립트의 핵심 기술입니다. 활성화하면 메모리를 과도하게 점유하는 웹 네이티브 데이터베이스 동기화 로직을 물리적으로 차단하고 스크립트가 가벼운 방식으로 관리합니다. 장점: 웹페이지를 연 지 몇 분 만에 버벅거리거나 브라우저가 충돌(OOM)하는 문제를 근본적으로 해결하여, 수만 개의 파일이 있는 클라우드도 데스크톱 소프트웨어처럼 매끄럽게 사용할 수 있게 해줍니다.

**Q: 왜 홈 화면의 My Pack에 “기본” 태그가 표시되며 즐겨찾기 추가, 복사, 이동, 이름 바꾸기 또는 삭제를 할 수 없나요?**
**A:** 이는 공식 보안 메커니즘입니다. 스크립트는 이 보호 기능 위에서 My Pack의 공유 제한만 해제했습니다.

**Q: 스크립트의 “파일 중복 확인”과 “폴더 중복 확인”의 본질적인 차이점은 무엇인가요? 실제 사용 시 어떻게 선택해야 하나요?**
**A:** 1. **파일 중복 확인:** “단일 파일” 단위를 대상으로 합니다. 해시 정밀 일치, 비디오 재생시간 유사도, 이름 유사도의 세 가지 모드 알고리즘을 사용하여 클라우드 곳곳에 흩어진 중복 파일(예: 중복 저장된 단일 에피소드 영상, 단일 이미지 등)을 정리하는 데 적합합니다. 2. **폴더 중복 확인:** “폴더 구조 전체”를 대상으로 합니다. 이름 일치, 내부 파일 중복도(유사도 일치), 부분 집합 중복(포함율 일치)의 세 가지 모드 알고리즘을 사용합니다. 폴더의 “내부 본질”을 꿰뚫어 보기 때문에, 중복 저장된 “전체 시리즈”, “전체 이미지 팩” 또는 “다계층 자료”를 정리하는 데 매우 적합합니다. 두 폴더의 이름이 완전히 다르더라도 내부 파일이 고도로 일치하거나 포함 관계가 있다면 정확하게 찾아낼 수 있습니다.

> **건의:** 낱개 파일 정리는 “파일 중복 확인”, 대형 리소스 팩 정리는 “폴더 중복 확인”을 권장합니다. 파일 중복 확인 중 정밀 일치는 해시 매칭을 사용하므로 안심하고 일괄 선택하여 삭제할 수 있습니다. 반면 파일 중복 확인의 재생시간/이름 유사도 및 폴더 중복 확인의 알고리즘은 유사도 기반이므로 오삭제 방지를 위해 일괄 선택 시 주의가 필요합니다.

**Q: 왜 “이미지로 검색” 시 때때로 이미지가 직접 업로드되지 않고 붙여넣으라는 메시지가 뜨나요?**
**A:** 이는 브라우저의 보안 정책(CORS 교차 출처 제한) 때문입니다. 스크립트가 클라우드 이미지를 검색 서버로 직접 전송할 수 없는 경우, 현재 화면의 스크린샷을 클립보드에 자동으로 복사합니다. 팝업된 검색 페이지에서 `[Ctrl+V]`를 누르면 즉시 인식이 완료됩니다.

**Q: 왜 스크립트로 비디오 재생 시 왜 소리만 나고 화면이 나오지 않나요?**
**A:** 보통 해당 비디오가 m3u8 다중 오디오/트랙 스트리밍 파일인 경우 발생합니다. 클라우드 인코딩 후 소리와 화면 트랙이 분리되었으나 웹 플레이어에서 이러한 특수 스트림의 온라인 다중 트랙 파싱을 지원하지 않기 때문입니다. 이 경우 우측 하단의 “외부 재생” 버튼을 클릭하여 로컬의 PotPlayer 등 전문 플레이어로 스트리밍 재생을 권장합니다.

**Q: 왜 내가 모바일 앱이나 공식 웹 사이트에서 방금 조작(업로드, 삭제, 이동 등)한 파일이 스크립트의 중복 확인 또는 분석 목록에 실시간으로 반영되지 않나요?**
**A:** 스크립트는 최상의 검색 및 분석 속도를 보장하기 위해 메모리 스냅샷 방식을 사용합니다. 매번 수만 개의 파일 정보를 다시 불러오지 않기 때문입니다. 해결 방법: 다른 클라이언트에서 파일을 많이 수정했다면 페이지를 새로고침하여 전체 인덱스를 다시 가져오세요. 스크립트 작업 중에는 여러 기기에서 동시에 데이터를 수정하는 것을 피하는 것이 좋습니다.

**Q: 일괄 압축 해제 시 여러 파일에 각각 다른 비밀번호를 한 번에 설정할 수 있나요?**
**A:** 네, 가능합니다. 스마트 매칭 로직: 알고 있는 비밀번호들을 “비밀번호 금고”에 미리 저장해 두면, 일괄 압축 해제 시 스크립트가 각 압축 파일마다 금고 속의 모든 비밀번호를 자동으로 시도합니다. 매칭에 성공하면 사람의 개입 없이 자동으로 해제가 시작됩니다.

**Q: 일괄 압축 해제에 실패하면 어떻게 하나요?**
**A:** 비밀번호 금고에 정확한 압축 해제 비밀번호를 입력했는지 확인해 주세요. 파일 자체가 손상된 경우 클라우드 서버에서도 압축 해제를 완료할 수 없습니다.

**Q: 내보낸 설정 JSON 파일에는 어떤 정보가 포함되나요?**
**A:** JSON 파일에는 인핸서 마스터에서 설정한 중복 확인 기본 설정, 블랙리스트/화이트리스트 규칙, 비밀번호 금고 등 로컬 설정 데이터만 포함됩니다. 사용자의 클라우드 계정 비밀번호는 **포함되지 않으므로** 안심하고 기기 간 이동 시 사용할 수 있습니다.

**Q: 왜 일괄 삭제 시 일부 파일이 보호됨으로 표시되거나 삭제되지 않나요?**
**A:** 해당 파일들이 리소스 관리자에 기록되어 있는지 확인해 주세요. 설정에서 “삭제 시 관리 도구 기록 리소스 건너뛰기”를 활성화했다면 스크립트가 해당 파일을 보호 항목으로 간주하여 오삭제를 방지합니다. 해결 방법: 1. 설정에서 해당 보호 규칙을 비활성화한 후 삭제하거나, 2. 툴바/사이드바의 “리소스 관리자” 메뉴로 들어가 “정리 즉시 실행”을 클릭하여 해당 기록 항목을 강제로 물리 삭제하세요.

* **데이터 로컬화**: 일괄 이름 바꾸기, 파일 분석, 압축 해제 등 모든 핵심 기능은 브라우저를 통해 공식 API와 직접 통신합니다. 사용자의 토큰, 비밀번호 금고 및 데이터는 **로컬 브라우저에만 저장**됩니다.

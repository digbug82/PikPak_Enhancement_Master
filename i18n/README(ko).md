## ✨ 주요 기능

### ✨ 경험 및 내비게이션 엔진

* **사이드 버튼 내비게이션**: 마우스 사이드 버튼으로 앞으로 / 뒤로 이동하며 서로 다른 디렉토리 단계 사이를 빠르게 전환할 수 있습니다.
* **고급 경로 표시줄**: 휠 스크롤, 같은 단계 디렉토리 드롭다운 전환, 경로 표시 및 원본 추적 점프를 지원합니다.
* **전체 드라이브 검색 강화**: 파일명과 파일 경로로 검색하고, 키워드 일치 부분을 빨간색으로 표시하며, 긴 파일명은 일치 키워드 주변 내용을 우선 표시합니다.
* **탐색 환경설정**: 정렬 환경설정, 보기 환경설정, 탐색 위치 유지, 미디어 모드, 버튼 텍스트 숨김, 미디어 커버 썸네일 흐림 및 야간 모드를 지원합니다.

### 🌐 전송, 클라우드 다운로드 및 공유 분석

* **공유 파일 투시**: 공유 콘텐츠를 재귀적으로 스캔하고 필터링하여 저장 전에 내부 구조를 미리 볼 수 있습니다.
* **공유 관리**: 공유 추출 횟수 상한을 설정할 수 있으며, 조건에 도달하면 공유를 자동 취소해 링크를 무효화할 수 있습니다.
* **업로드 보호**: 업로드 중 페이지 이탈 알림을 내장하고, 업로드 중단, 잔여 작업, 비정상 파일을 보호 및 정리합니다.
* **클라우드 다운로드 강화**: 일괄 오프라인 링크 자동 중복 제거를 지원하며, 내장 마그넷 스마트 정리로 Base32 / Hex 해시를 추출하고 불필요한 텍스트를 제거할 수 있습니다.
* **TXT 마그넷 추출**: TXT 텍스트를 미리 보고, 텍스트에서 마그넷, HTTP, FTP, ed2k, thunder 등 다운로드 링크를 인식해 클라우드 다운로드로 제출할 수 있습니다.
* **마그넷 미리보기**: 마그넷 콘텐츠에 여러 미리보기 이미지가 있을 때 다중 이미지 미리보기를 지원합니다.

### 🎬 몰입형 미디어 강화

* **재생 엔진 강화**: 0.5x - 3.0x 배속, 회전 및 뒤집기, 강제 비율, 오프닝 / 엔딩 자동 건너뛰기, 연속 재생 / 반복 및 진행 막대 썸네일 미리보기를 지원합니다.
* **음악 플레이어**: 오디오 파일 온라인 재생, 재생목록, 순차 재생, 무작위 재생, 단일 곡 반복, 팝업 모드, 커버 표시 및 볼륨 제어를 지원합니다.
* **오디오 팝업 모드**: 음악 플레이어를 작은 창으로 분리하고 재생 제어, 커버 표시, 테마 적응을 유지할 수 있습니다.
* **자막 시스템**: 클라우드의 같은 이름 자막 로드, 로컬 자막 가져오기, 온라인 자막 검색을 지원하며, 자막 싱크 오프셋, 위치, 글꼴 크기, 배경 투명도를 조정할 수 있습니다.
* **시각 보조**: 이미지 또는 비디오 현재 프레임으로 이미지 검색을 수행해 커버, 배우, 애니메이션 또는 소재 출처를 역추적할 수 있습니다.
* **미디어 모드**: 설정에서 미디어 모드를 활성화하면 비디오 전용 또는 이미지 전용 디렉토리를 기본적으로 이름 A-Z 순으로 정렬해 에피소드 / 만화 탐색의 연속성을 높일 수 있습니다.

### ⚡ 다운로드, 외부 재생 및 배포

* **기본 열기 방식**: 스크립트 플레이어와 PotPlayer 중 기본 재생 방식을 선택할 수 있습니다.
* **PotPlayer 프로토콜 복구 도우미**: 내장 PotPlayer 프로토콜 감지 및 복구 보조로 브라우저가 외부 플레이어를 호출하지 못하는 문제를 줄입니다.
* **RPC 배포**: RPC 프로토콜을 통해 파일을 Aria2 / Motrix 같은 다운로드 노드로 보낼 수 있습니다.
* **디렉토리 구조 복원**: 전체 폴더를 전송할 때 클라우드 드라이브의 트리 구조를 자동으로 복원하여 다운로드 후 디렉토리가 평면화되는 것을 방지합니다.
* **다운로드 필터**: 파일 크기, 확장자, 이름 키워드로 파일을 필터링하며 브라우저 다운로드와 Aria2 / Motrix 전송에 적용됩니다.
* **다운로드 직접 링크 가속 도메인**: 사용자 지정 다운로드 가속 도메인을 지원하며, 접두사 전체 원본 링크와 URL 매개변수 전달 두 가지 리버스 프록시 재작성 모드를 제공합니다.
* **예외 처리**: 전송 시 0KB 무효 파일을 자동으로 건너뛰고 실패 목록 내보내기를 지원합니다.

## 📥 설치 가이드

1. **스크립트 관리자 설치**: [Violentmonkey](https://violentmonkey.github.io/get-it/) 설치를 우선 권장하며, [Tampermonkey](https://www.tampermonkey.net/)도 사용할 수 있습니다.
2. **스크립트 설치**: **[지금 설치](https://github.com/digbug82/PikPak_Enhancement_Master/raw/main/PikPak_Enhancement_Master.user.js)**를 클릭합니다.
3. **PikPak 웹 열기**: [PikPak](https://mypikpak.com/drive)에 접속하고 계정에 로그인합니다.
4. **Enhancement Master 시작**:
   * 일반 모드: 로그인 후 사이드바에 떠 있는 파란색 **PikPak Logo 공**을 클릭해 진입합니다.
   * 초고속 모드: 로그인 후 스크립트가 자동으로 켜져 웹 클라이언트를 인계할 수 있습니다.

## ❓ 자주 묻는 질문

**Q: 스크립트를 설치했는데 파란색 플로팅 공이 표시되지 않는 이유는 무엇인가요?**  
**A:** 다음 순서로 확인하세요:

1. 스크립트 관리자에서 **PikPak Enhancement Master**가 켜져 있는지 확인합니다.
2. PikPak 웹 계정에 로그인했는지 확인합니다.
3. **Tampermonkey**를 사용하는 경우 브라우저 확장 프로그램 상세 페이지의 **"사용자 스크립트 허용"**이 체크되어 있는지 확인합니다.
4. 최신 버전의 **Edge / Chrome** 브라우저 사용을 권장합니다.
5. 위 단계를 완료해도 플로팅 공이 표시되지 않으면 다른 모든 브라우저 확장을 임시로 끈 뒤 페이지를 새로고침(F5)하고 다시 시도하세요.

**Q: 여러 단계의 폴더에서 이전 단계로 돌아가기 번거로운데, 꼭 메인 화면에서 다시 들어가야 하나요?**  
**A:** 아닙니다. 스크립트에서 경로 표시줄이 있는 모든 페이지는 **마우스 휠 가로 스크롤**을 지원합니다.

디렉터리 단계가 깊어 앞쪽 상위 폴더 경로가 숨겨졌다면, 마우스를 상단 경로 표시줄 영역으로 옮긴 뒤 휠을 스크롤하세요. 경로 표시줄이 가로로 스크롤되며 숨겨진 상위 경로가 표시됩니다.

표시된 뒤 해당 상위 폴더 이름을 바로 클릭하면 이전 단계로 빠르게 돌아가거나 원하는 상위 디렉터리로 이동할 수 있습니다.

**Q: 파일 중복 확인 / 폴더 중복 확인 시 체크된 파일은 원본 하나를 자동으로 남기나요, 아니면 목록에서 다시 필터링해야 하나요?**  
**A:** 파일 중복 확인과 폴더 중복 확인의 체크 항목은 본질적으로 **실제 작업 항목**입니다. 중복 확인 모드 자체는 파일을 바로 삭제하지 않고, 중복 가능성이 있는 항목만 나열합니다. 사용자가 확인하고 체크한 뒤 삭제를 클릭해야 실제로 정리됩니다.

중복 확인 결과에서:

* 각 그룹 헤더의 체크박스는 해당 그룹 전체 선택을 의미합니다;
* 그룹 내 정렬은 시간, 크기, 경로, 이름 등의 규칙에 따라 표시 순서만 다시 배열하며, 무엇을 삭제하고 무엇을 남길지 자동으로 결정하지 않습니다;
* **일괄 선택**을 클릭했을 때만 스크립트가 규칙에 따라 자동 처리합니다. 각 그룹에서 1개를 남기고 나머지 항목이 체크됩니다.

또한 파일 중복 확인의 **폴더별 선택**은 중복 목록에서 특정 폴더 안의 파일을 체크합니다. **선택 반전**은 해당 폴더 밖의 파일을 체크합니다.

주의할 점은, 파일 중복 확인의 **정밀 일치**는 해시값 일치를 사용하므로 보통 일괄 선택과 함께 정리하기에 더 적합합니다. 파일 중복 확인의 **재생시간 유사 / 이름 유사** 및 폴더 중복 확인은 모두 유사도 알고리즘 기반이므로 삭제 전 수동 확인을 권장합니다.

**Q: 스크립트의 "정밀 일치"는 중복 파일이 없다고 하는데 클라이언트에서는 중복 파일이 보입니다. 새로고침하거나 브라우저를 바꿔도 스캔되지 않으면 어떻게 해야 하나요?**  
**A:** 스크립트의 "정밀 일치"는 **파일 콘텐츠 해시**로 판단하며 파일명으로 판단하지 않습니다. 파일 해시값과 파일 크기가 모두 같을 때만 정밀 중복으로 판정됩니다.

클라이언트에 표시되는 "중복 파일"이 반드시 내용이 완전히 같은 파일을 뜻하지는 않습니다. 같은 이름의 파일, 파일명 끝에 `(1)` / `(2)`가 붙은 파일, 또는 크기나 재생시간이 비슷한 파일일 수 있습니다. 실제 내용이 다르면 스크립트의 "정밀 일치"는 이를 중복으로 분류하지 않습니다.

이런 "겉보기 중복" 파일을 찾고 싶다면 파일 중복 확인 또는 폴더 중복 확인의 **이름 유사** 모드를 사용할 수 있습니다.

**Q: 일부 비디오는 처음 열거나 화질을 전환할 때 재생할 수 없다고 표시되지만, 잠시 후 다시 열면 재생되는 이유는 무엇인가요?**  
**A:** 이런 상황은 보통 PikPak 공식 비디오 스트림의 준비 상태 또는 브라우저 디코딩 상태와 관련이 있으며, 반드시 스크립트 자체의 판단 오류라고 볼 수는 없습니다.

일부 비디오는 공식 웹에서도 처음에는 열리지 않거나, 화질 전환 직후 잠시 실패했다가 나중에 다시 열면 재생되는 경우가 있습니다. 스크립트가 받은 하위 오류는 일반적으로 다음과 비슷합니다:

```text
[VideoError] Code: 4, Msg: PipelineStatus::DEMUXER_ERROR_COULD_NOT_OPEN: FFmpegDemuxer: open context failed
```

이 오류는 "공식 리소스가 아직 준비 중"인지, 아니면 "현재 화질이 정말 재생 불가"인지 안정적으로 구분할 수 없습니다. 스크립트가 이를 강제로 "준비 중"으로 보고 계속 재시도하면, 실제로 재생할 수 없는 비디오까지 재생 가능하다고 오판해 더 나쁜 경험을 만들 수 있습니다.

따라서 스크립트는 현재 "현재 해상도/화질을 사용할 수 없음" 안내를 유지하여 사용자를 오도하지 않도록 합니다.

이런 상황이 발생하면 다음을 시도해 볼 수 있습니다:

* 몇 초 기다린 뒤 비디오를 다시 엽니다;
* 다른 화질로 전환합니다;
* 외부 재생으로 재생을 시도합니다.

나중에 공식의 더 신뢰할 수 있는 상태 필드를 찾으면 이런 오탐 문제를 계속 최적화할 수 있습니다.

**Q: 일괄 압축 해제 중 "시스템 사용 중 / 재시도 대기"가 표시됩니다. 스크립트 버그인가요?**  
**A:** 아닙니다. 이는 보통 PikPak 클라우드 서버가 동시 처리 또는 빈도 제한을 트리거했기 때문이며, 공식 인터페이스가 현재 바쁜 상태라는 뜻이지 스크립트 자체의 이상은 아닙니다.

특정 파일이 여러 번 재시도해도 계속 실패한다면, 해당 리소스가 클라우드에서 손상되었거나 제한되었거나 현재 서버 상태가 좋지 않을 수 있습니다. 나중에 다시 시도하거나 파일을 로컬로 다운로드한 뒤 처리하는 것을 권장합니다.

**Q: 일괄 삭제 후 복원했더니 파일 수는 정상인데 원래 폴더로 돌아가지 않았고 디렉터리 구조가 엉킨 것 같습니다. 한 번에 복구할 수 있나요?**  
**A:** 이런 상황은 대개 작업 오류도 아니고 스크립트 로직이 파일을 잃어버린 것도 아닙니다. PikPak 공식 휴지통이 **가상 경로 / 집계 결과**에서 복원을 수행할 때 나타나는 동작에 가깝습니다.

파일 중복 확인, 파일 투시, 폴더 투시, 최근 항목, 즐겨찾기 같은 **집계 보기 또는 가상 경로**에서 파일이 일괄 삭제된 경우, 해당 파일들은 서로 다른 원본 폴더에서 온 것일 수 있습니다. 삭제 후 휴지통에서 복원할 때 공식 복원 메커니즘이 각 항목을 원래 부모 디렉터리로 하나씩 복원하지 않고, 많은 파일을 하나의 통합 디렉터리로 모아 복원해 원래 다단계 디렉터리 구조가 깨질 수 있습니다.

파일 중복 확인, 파일 투시, 폴더 투시, 최근 항목, 즐겨찾기 같은 집계 결과에서 일괄 삭제하기 전에 해당 파일이 정말 더 이상 필요하지 않은지 먼저 확인하세요. 특히 파일이 여러 디렉터리에서 온 경우, 삭제 후 휴지통 일괄 복원에 의존하지 않는 것이 좋습니다.

**Q: 파일을 일괄 삭제할 때 일부 파일이 보호되었거나 삭제할 수 없다고 표시되는 이유는 무엇인가요?**  
**A:** 먼저 해당 파일이 **리소스 관리자**에 기록되어 있는지 확인하세요. 설정에서 "삭제 시 관리 도구 기록 리소스 건너뛰기"를 켠 경우, 스크립트는 일치한 파일을 보호 항목으로 간주해 실수로 삭제되는 것을 방지합니다.

**해결 방법:**

1. 설정에서 해당 보호 규칙의 체크를 해제한 뒤 다시 삭제를 실행합니다;
2. 또는 도구 모음 / 사이드바의 **"리소스 관리자"** 진입점을 클릭하고 **"정리 즉시 실행"**을 선택해 해당 기록 항목을 강제 물리 정리합니다.

**Q: "붙여넣기" 작업을 실행한 뒤 파일이 목록에 표시되지 않는 이유는 무엇인가요?**  
**A:** 먼저 클라우드 드라이브의 남은 공간이 충분한지 확인하세요. PikPak 공식은 현재 "공간 부족"에 대해 보통 **무음 차단** 전략을 사용합니다. 즉, 용량을 초과해도 명확한 안내가 뜨지 않고 백그라운드에서 작업이 바로 중단될 수 있습니다.

따라서 붙여넣기 후 파일이 나타나지 않는 가장 흔한 이유는 용량 제한이 트리거되었기 때문입니다. 먼저 공간을 정리한 뒤 다시 시도하세요.

**Q: "다중 계정 데이터 마이그레이션"이란 무엇이며, 어떻게 사용하나요?**  
**A:** 이 기능은 현재 계정의 파일 또는 폴더를 사용자의 다른 PikPak 계정으로 빠르게 전송할 수 있습니다.

**작업 단계는 다음과 같습니다:**

1. 현재 계정에서 마이그레이션할 파일 또는 폴더를 선택합니다;
2. 하단의 **"데이터 마이그레이션"** 버튼을 클릭합니다;
3. 스크립트가 자동으로 데이터를 암호화해 패키징하고 현재 계정에서 로그아웃합니다;
4. 이후 대상 계정으로 정상 로그인하면 됩니다;
5. 로그인에 성공하면 스크립트가 로컬 캐시의 마이그레이션 패키지를 자동으로 감지하고, 한 번의 클릭으로 수신하라는 팝업을 표시합니다.

## 🛡️ 개인정보 및 보안 안내

* **로컬 우선**: 이 스크립트의 모든 핵심 기능은 브라우저를 통해 PikPak 공식 API와 직접 상호작용합니다. 계정 Token, 비밀번호 금고, 대부분의 로컬 설정 데이터는 기본적으로 로컬 브라우저 환경에 저장됩니다.
* **무수집**: 스크립트는 사용자 개인정보를 **능동적으로 수집하지 않으며**, 사용자의 파일 정보나 계정 인증 정보를 어떤 제3자 서버에도 **절대 업로드하지 않습니다**.
* **제3자 인터페이스**: "온라인 자막 검색" 또는 "이미지로 검색" 같은 확장 기능을 사용할 때만 필요한 검색 키워드 또는 이미지 특징 파라미터를 관련 공공 서비스로 전송하며, 개인 신원 정보는 포함되지 않습니다.

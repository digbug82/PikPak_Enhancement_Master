## ✨ 주요 기능 (Features)

### ✨ 경험 및 탐색 엔진

* **인터랙션 재구성**: 인터페이스를 Windows 파일 탐색기 스타일로 재구성하여 전반적인 조작 경험을 향상시켰습니다.
* **초고속 모드**: 순정 웹 로직을 대체하여 기본 동기화를 차단합니다. (**설정에서 활성화 권장**)
* **고급 경로 표시줄**: 마우스 휠 스크롤, 드롭다운 메뉴 전환, 경로 복원 및 상위 이동을 지원합니다.
* **백그라운드 자동 인덱싱**: 홈 아이콘의 파란색 점멸을 통해 디렉토리 트리 동기화 상태를 알려줍니다.
* **사용자 경험 향상**: 다크 모드, 비디오 재생 시간 표시, 원클릭 썸네일 흐리게(프라이버시 모드), 즐겨찾기 및 파일 유형 정렬을 지원합니다.

### 🌐 전송 및 공유 센터

* **공유 관리**: 추출 횟수 제한을 지원하며, 횟수 도달 시 공유가 자동 취소됩니다.
* **오프라인 전송**: 일괄 오프라인 작업 및 링크 내보내기 지원.
* **초고속 업로드**: 대용량 파일 업로드 지원 및 소용량 파일 업로드 실패율을 대폭 낮췄습니다.
* **클라우드 다운로드 향상**: 일괄 링크 추가 시 **자동 중복 제거**; 내장형 **마그넷 스마트 정리 엔진**을 통해 깨진 글자나 노이즈 텍스트에서 마그넷을 자동 추출하고 Base32 디코딩을 지원합니다.

## ❓ 자주 묻는 질문 (FAQ)

**Q: 스크립트를 설치했는데 왜 파란색 떠 있는 구슬이 표시되지 않나요?**  
**A:** PikPak 웹 버전에 로그인했는지 확인해 주세요. 로그인했는데도 표시되지 않으면 웹페이지를 새로고침(F5)해 보거나, 광고 차단 플러그인(예: AdGuard / uBlock Origin)에 의해 차단되었는지 확인하여 플러그인을 일시적으로 끄거나 이 사이트를 화이트리스트에 추가해 보세요.

**Q: "초고속 모드"란 무엇이며, 왜 활성화해야 하나요?**  
**A:** 활성화하면 스크립트가 전체 화면으로 웹 UI를 대체하고 순정 웹 기능을 완전히 제어합니다. 이 모드는 메모리를 심하게 소모하는 순정 동기화 로직을 물리적으로 차단하여 대량의 파일 환경에서의 렉과 충돌 문제를 근본적으로 해결합니다. **주의:** 초고속 모드에서는 최소화 및 닫기 버튼이 자동으로 숨겨집니다(설정에서 체크를 해제해야 원래 웹으로 돌아갈 수 있습니다). **권장:** 파일이 많을 경우 최고의 응답 속도와 부드러운 경험을 위해 이 모드를 켜는 것을 강력히 권장합니다.

**Q: 홈 화면의 My Pack 폴더에 "기본" 태그가 표시되고 즐겨찾기, 복사, 이동, 이름 바꾸기, 삭제를 할 수 없는 이유는 무엇인가요?**  
**A:** 이는 공식적인 보안 보호 메커니즘입니다. 스크립트는 이를 바탕으로 기본 폴더인 My Pack의 공유 제한만을 해제했습니다.

**Q: 스크립트의 "파일 중복 확인"과 "폴더 중복 확인"의 본질적인 차이는 무엇이며, 어떻게 선택해야 하나요?**  
**A:** **파일 중복 확인:** "단일 파일" 단위입니다. 정밀 일치(해시), 재생시간 유사, 이름 유사 3가지 모드 알고리즘을 사용하며, 클라우드 드라이브에 흩어져 있는 중복 파일(예: 중복 저장된 단일 에피소드, 단일 이미지)을 정리하는 데 적합합니다.<br>
**폴더 중복 확인:** "폴더의 전체 구조" 단위입니다. 이름 일치, 내부 파일 중복도(유사도 일치), 부분집합 중복(포함율 일치) 3가지 모드 알고리즘을 사용합니다. 폴더의 "내부 본질"을 꿰뚫어 보아 중복 저장된 "전체 시리즈", "전체 이미지 팩" 또는 "다중 계층 자료"를 정리하는 데 매우 적합합니다. 두 폴더의 이름이 완전히 다르더라도 내부 파일이 고도로 일치하거나 포함 관계가 있다면 정확하게 찾아냅니다.

> **권장:** 파일 중복 확인의 "정밀 일치"는 해시 기반이므로 일괄 선택 및 삭제가 안전합니다. 파일 중복 확인의 재생시간/이름 유사도 및 폴더 중복 확인의 세 가지 모드는 유사성 기반의 매칭이므로 실수로 삭제하지 않도록 일괄 선택 전 직접 확인하시기 바랍니다.

**Q: 왜 갈수록 중복 파일이 더 많이 발견되나요? 전체 인덱스 데이터가 유실된 건가요?**  
**A:** 안심하세요. 데이터 유실이나 오류가 아니며, 중복 결과 중 "정밀 일치"의 개수는 항상 안정적입니다. 결과가 많아지는 것은 비디오 길이를 기반으로 한 "재생시간 유사" 매칭이 더욱 완벽해졌기 때문입니다. PikPak의 원본 API는 항상 비디오 길이를 직접 제공하지 않아서, 초기에는 데이터 누락으로 인해 많은 중복 비디오가 건너뛰어집니다. 이를 해결하기 위해 스크립트에는 비디오 메타데이터를 백그라운드에서 비동기적으로 스니핑하여 정확한 재생 시간을 추출하고 브라우저 캐시에 영구 저장하는 알고리즘이 내장되어 있습니다. 사용 시간이 늘어남에 따라 로컬의 재생시간 데이터베이스가 점점 더 완벽해지며, 엔진은 보완된 데이터를 기반으로 파일 이름은 다르지만 내용 길이가 동일한 숨겨진 중복 리소스를 더 많이 찾아냅니다. 따라서 결과가 점점 더 많아지고 정확해집니다.

**Q: "이미지로 검색"을 할 때 왜 이미지가 직접 업로드되지 않고 붙여넣으라고 하나요?**  
**A:** 브라우저 보안 정책(CORS 교차 도메인 제한) 때문입니다. 스크립트가 클라우드 이미지를 검색 서버로 직접 전송할 수 없을 때, 현재 화면의 캡처본을 클립보드에 자동으로 복사합니다. 팝업된 검색 페이지에서 `[Ctrl+V]`를 누르기만 하면 인식을 완료할 수 있습니다.

**Q: 스크립트로 비디오를 재생할 때 왜 소리만 나고 화면은 안 나오나요?**  
**A:** 이는 대개 해당 비디오가 m3u8 다중 오디오/비디오 트랙 스트리밍 파일로, 클라우드 인코딩 후 소리와 화면 트랙이 분리되었기 때문입니다. 웹 플레이어는 아직 이러한 특수 비디오 스트림의 온라인 다중 트랙 파싱을 지원하지 않습니다. 오른쪽 하단의 "외부 재생" 버튼을 클릭하여 PotPlayer 등 전문 플레이어를 로컬에서 직접 호출해 스트리밍하는 것을 권장합니다.

**Q: 해상도 메뉴의 "원본 (고속)"과 일반 "원본 화질"의 차이는 무엇인가요?**  
**A:** 화질은 거의 완벽하게 일치하며, 주요 차이점은 기반 디코딩 및 전송 프로토콜에 있습니다. "원본 (고속)"은 스트리밍에 최적화된 공식 링크를 사용합니다. PotPlayer 등의 외부 플레이어를 호출할 때 "고속"이 포함된 링크를 사용하면 가장 빠른 로딩 응답과 가장 부드러운 탐색 경험을 얻을 수 있습니다. 웹 재생이든 외부 스트리밍이든 이 옵션을 우선적으로 선택하는 것을 권장합니다.

**Q: 스마트폰 앱이나 공식 웹에서 방금 수행한 작업(예: 업로드, 삭제, 이동)이 왜 스크립트의 분석 목록에 실시간으로 동기화되지 않나요?**  
**A:** 스크립트는 최고의 검색 및 분석 속도를 보장하기 위해 메모리 스냅샷 메커니즘을 사용하여 매번 수만 개의 파일 정보를 새로 가져오지 않습니다. 해결 방법: 다른 클라이언트에서 너무 많은 파일을 수정한 경우 웹페이지를 새로고침하여 전체 인덱스를 다시 가져오세요. 스크립트 조작 중에는 여러 기기에서 동시에 데이터를 수정하지 않는 것이 좋습니다.

**Q: 일괄 압축 해제 시 여러 파일에 한 번에 다른 암호를 설정할 수 있나요?**  
**A:** 가능합니다. 스마트 매칭 로직: 사용자가 알고 있는 암호를 "비밀번호 금고"에 미리 저장해 두기만 하면, 스크립트가 일괄 압축 해제 시 각각의 압축 파일에 대해 금고의 모든 암호를 자동으로 대입해 봅니다. 일치하면 사람의 개입 없이 자동으로 해제를 시작합니다.

**Q: 일괄 압축 해제 중 "시스템이 바쁩니다. 재시도 중"이라고 뜨는데 스크립트 버그인가요?**  
**A:** 아닙니다. 이는 대개 PikPak 클라우드 서버의 동시 처리 빈도 제한을 초과했기 때문입니다. 이 메시지가 표시되면 클라우드 공식 API도 현재 혼잡한 상태임을 의미합니다. 파일이 여러 번 재시도해도 실패한다면 리소스가 클라우드에서 손상되었거나 제한된 상태일 수 있으므로 나중에 다시 시도하거나, 다시 업로드/로컬 다운로드하여 처리하는 것을 권장합니다.

**Q: 내보낸 백업 JSON 설정 파일에는 어떤 정보가 포함되나요?**  
**A:** JSON 파일에는 인핸서 마스터에서 설정한 중복 확인 선호도, 블랙/화이트리스트 규칙, 비밀번호 금고 등의 로컬 구성 데이터만 포함되며, 계정 및 비밀번호는 **포함되지 않으므로** 안심하고 기기 간 마이그레이션에 사용할 수 있습니다.

**Q: 백업을 가져오면 기존 설정이나 명단이 덮어쓰여지나요?**  
**A:** 아닙니다. 스크립트는 "스마트 병합" 로직으로 업그레이드되어 가져오기 시 리소스 관리자, 기록 등의 목록 항목을 로컬 데이터와 자동으로 병합 및 중복 제거합니다. 기본 설정 항목(예: 검색 엔진, Aria2 주소 등)만 가져온 파일 기준으로 덮어쓰여져, 로컬에 누적된 데이터가 손실되지 않도록 보장합니다.

**Q: 파일을 일괄 삭제할 때 일부 파일이 보호되거나 삭제할 수 없다는 메시지가 나오는 이유는 무엇인가요?**  
**A:** 해당 파일들이 리소스 관리자에 기록되어 있는지 확인해 보세요. 설정에서 "삭제 시 관리 도구 기록 리소스 건너뛰기"를 켰다면 스크립트가 일치하는 파일을 보호 대상으로 간주하여 오삭제를 방지합니다.<br>
해결 방법: 
1. 설정에서 해당 보호 규칙의 체크를 해제한 후 다시 삭제를 진행합니다.
2. 또는 도구 모음/사이드바의 "리소스 관리자"로 들어가 "정리 즉시 실행"을 클릭하여 해당 기록 항목들을 강제로 영구 삭제합니다.

**Q: "붙여넣기" 작업을 실행했는데 왜 목록에 파일이 나타나지 않나요?**  
**A:** 클라우드 드라이브의 남은 용량이 이 파일들을 저장하기에 충분한지 확인해 주세요. PikPak은 현재 "공간 부족" 시 **조용한 차단** 정책을 사용하여, 제한 용량을 초과해도 구체적인 오류 팝업을 띄우지 않고 백그라운드에서 바로 작업을 중지합니다. 붙여넣기 후 파일이 나타나지 않으면 대개 용량 제한에 도달했기 때문이므로 공간을 확보한 후 다시 시도해 보세요.

**Q: 클라우드 다운로드 창의 "차단 방지 마그넷 자동 복구"는 어떤 기능이며, 언제 켜는 것을 권장하나요?**  
**A:** 이 기능은 링크에 섞인 한자, 이모지, 기호 등 텍스트 노이즈를 자동으로 제거하고 특징 코드를 지능적으로 식별하여 표준 형식으로 복원합니다.<br>
**권장: 이 기능은 기본으로 체크된 상태를 유지하는 것을 권장합니다.** SNS에서 얻은 수많은 관련 없는 문자가 섞이거나 앞부분이 잘린 오염된 마그넷을 자동으로 처리할 수 있으며, `magnet:?`로 시작하는 **표준 마그넷**은 자동으로 수정을 건너뛰고 정상 다운로드됩니다.

**Q: Aria2/Motrix 또는 브라우저로 폴더를 다운로드할 때 폴더의 계층 구조가 유지되나요?**  
**A:** **Aria2/Motrix는 계층 구조를 유지합니다.** 스크립트는 디렉토리 계층 클론 기술을 사용하여, 다운로드 후 로컬 하드 드라이브에 클라우드의 트리 구조를 완벽하게 재현합니다. 또한 Windows에서 지원하지 않는 특수 문자(예: `:`, `*`, `?` 등)를 폴더 이름에서 자동 제거하고, 다운로드 오류가 나지 않도록 중복 경로를 스마트하게 축소합니다.  
**브라우저 다운로드는 계층 구조를 유지하지 않습니다.** 브라우저 기본 다운로드 프로토콜의 한계로 인해 폴더 내의 모든 파일이 단일 디렉토리에 평면적으로 다운로드됩니다. **권장:** 복잡한 폴더 구조를 유지해야 하는 경우 반드시 Aria2 전송을 우선적으로 선택하세요.

## 🛡️ 개인정보 보호 및 보안 정책 (Privacy & Security)

* **데이터 로컬화**: 본 스크립트의 모든 핵심 기능(일괄 이름 바꾸기, 파일 분석, 압축 해제 등)은 사용자의 브라우저를 통해 직접 PikPak 공식 API와 상호 작용합니다. 사용자의 계정 토큰, 비밀번호 금고 및 파일 데이터는 **오직 로컬 브라우저에만 저장**됩니다.
* **수집 제로**: 스크립트는 사용자의 어떠한 개인정보 데이터도 수집하지 **않으며**, 파일 정보나 계정 자격 증명을 타사 서버로 절대 업로드하지 않습니다.
* **타사 API**: "자막 검색"(온라인) 또는 "이미지로 검색"과 같은 확장 기능을 사용할 때만, 스크립트는 관련 공공 서비스 API에 필요한 검색 키워드나 이미지 특징 파라미터를 전송하며, 이는 어떠한 개인 식별 정보도 포함하지 않습니다.

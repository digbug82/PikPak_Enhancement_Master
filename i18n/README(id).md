## Fitur Utama

### Peningkatan Pengalaman dan Navigasi

* **Navigasi tombol samping**: mendukung tombol samping mouse untuk maju / mundur, sehingga mudah berpindah cepat antar tingkat folder.
* **Bilah jalur lanjutan**: mendukung gulir roda mouse, perpindahan folder sejajar melalui dropdown, tampilan ulang jalur, dan lompatan penelusuran sumber.
* **Peningkatan pencarian seluruh drive**: mendukung pencarian berdasarkan nama file dan jalur file, menandai sebagian kata kunci dengan warna merah, dan untuk nama file panjang akan memprioritaskan tampilan di sekitar kata kunci yang cocok.
* **Preferensi penjelajahan**: mendukung preferensi pengurutan, preferensi tampilan, mempertahankan posisi jelajah, mode media, menyembunyikan teks tombol, memburamkan thumbnail sampul media, dan mode malam.

### Bookmark Tautan dan Arsip Cloud

* **Bookmark Tautan**: Bookmark Tautan menyimpan tautan bookmark, data sinkronisasi skrip, tautan magnet Arsip Cloud, dan konfigurasi cloud.
* **Arsip Cloud**: Mendukung pengarsipan tautan magnet yang dapat dilacak dari file terpilih ke “Bookmark Tautan > Arsip Cloud”, serta dapat menghapus file sumber untuk menghemat ruang.
* **Pemulihan Parsing Arsip Cloud**: Tautan magnet yang diarsipkan dapat dikirim ulang sebagai tugas unduhan cloud, dan status pengiriman akan ditulis kembali.
* **Pembersihan Arsip Cloud**: Mendukung mengosongkan semua tautan Arsip Cloud, menghapus tautan yang telah dikirim, dan menghapus satu tautan arsip.
* **Sinkronisasi Konfigurasi Cloud**: Mendukung unggah konfigurasi skrip ke cloud, tarik dari cloud, dan hapus konfigurasi cloud, memudahkan sinkronisasi preferensi skrip antar perangkat.

### Transfer, Cloud Download, dan Parser Berbagi

* **Parser Berbagi**: mendukung parsing tautan berbagi dan melihat isi berbagi sebelum disimpan, tanpa memakai ruang cloud drive Anda sebelum penyimpanan.
* **Tinjauan file berbagi**: mendukung pemindaian rekursif, filter, dan pratinjau isi berbagi, cocok untuk memeriksa struktur internal berbagi sebelum disimpan.
* **Riwayat parsing berbagi**: dapat mencatat tautan berbagi dan sandi yang sudah berhasil diparse, sehingga mudah dibuka ulang dengan cepat.
* **Manajemen berbagi**: mendukung batas jumlah pengambilan berbagi. Setelah kondisi terpenuhi, berbagi dapat dibatalkan otomatis agar tautan tidak berlaku.
* **Unggah Instan**: Saat mengunggah file, jika server sudah memiliki catatan file yang sama, unggahan dapat langsung selesai untuk mengurangi waktu tunggu unggah berulang.
* **Perlindungan unggahan**: menyediakan pengingat saat meninggalkan halaman unggahan, serta perlindungan dan pembersihan untuk unggahan terputus, tugas tersisa, dan file abnormal.
* **Peningkatan Cloud Download**: mendukung deduplikasi otomatis untuk tautan offline massal. Pembersih magnet cerdas bawaan dapat mengekstrak hash Base32 / Hex dan menghapus teks pengganggu.
* **Ekstraksi magnet TXT**: mendukung pratinjau teks TXT dan mengenali tautan magnet, HTTP, FTP, ed2k, thunder, dan tautan unduhan lain dari teks untuk dikirim ke Cloud Download.
* **Pratinjau magnet**: saat informasi pratinjau publik tersedia, dapat menampilkan jumlah file, ukuran total, jenis, hash, dan beberapa gambar pratinjau.

* **Peningkatan mesin pemutar**: mendukung kecepatan 0.5x - 3.0x, rotasi / balik, rasio paksa, lewati intro / outro otomatis, putar beruntun / loop, dan pratinjau thumbnail pada bilah progres.
* **Metode buka default**: dapat memilih metode pemutaran default antara pemutar skrip dan PotPlayer, serta mendukung pengaturan kualitas video default / kualitas pemutaran eksternal.

* **Mode popup audio**: mendukung pemisahan Pemutar musik menjadi jendela kecil sambil mempertahankan kontrol pemutaran, tampilan sampul, dan penyesuaian tema.

* **Asisten perbaikan protokol PotPlayer**: menyediakan deteksi dan bantuan perbaikan protokol PotPlayer bawaan, mengurangi masalah browser tidak dapat memanggil pemutar eksternal.

## Tanya Jawab Umum

**Q: Mengapa bola mengambang biru tidak muncul setelah skrip dipasang?**  
**A:** Periksa dengan urutan berikut:

1. Pastikan **PikPak Enhancement Master** aktif di pengelola userscript.
2. Pastikan Anda sudah masuk ke akun PikPak Web.
3. Jika menggunakan **Tampermonkey**, pastikan **"Allow User Scripts"** di halaman detail ekstensi browser sudah dicentang.
4. Disarankan menggunakan browser **Edge / Chrome** versi terbaru.
5. Jika setelah langkah di atas bola mengambang tetap tidak muncul, nonaktifkan sementara semua ekstensi browser lain, lalu muat ulang halaman (F5) dan coba lagi.

**Q: Kembali dari folder bertingkat terasa merepotkan. Apakah harus masuk ulang dari halaman utama?**  
**A:** Tidak perlu. Semua halaman dalam skrip yang memiliki bilah jalur mendukung **gulir horizontal dengan roda mouse**.

Saat tingkat direktori cukup dalam dan jalur folder induk di bagian depan tersembunyi, arahkan mouse ke area bilah jalur atas, lalu gulir roda mouse. Bilah jalur akan bergulir horizontal dan menampilkan jalur induk yang tersembunyi.

Setelah muncul, klik langsung nama folder induk yang sesuai untuk kembali cepat ke tingkat sebelumnya atau melompat ke direktori induk mana pun.

**Q: Saat Cari Duplikat File / duplikat folder, apakah file yang tercentang otomatis menyisakan satu file asli, atau saya harus menyaring lagi di daftar?**  
**A:** Item yang dicentang di Cari Duplikat File dan duplikat folder pada dasarnya adalah **target operasi nyata**. Mode pencarian duplikat tidak langsung menghapus file; mode ini hanya menampilkan item yang mungkin duplikat. Anda perlu memeriksa dan memilihnya, lalu klik hapus untuk benar-benar membersihkan.

Di hasil duplikat:

* Kotak centang pada header setiap grup berarti memilih seluruh grup;
* Pengurutan dalam grup hanya mengubah urutan tampilan berdasarkan waktu, ukuran, jalur, nama, dan aturan lain. Itu tidak otomatis menentukan mana yang dihapus atau disimpan;
* Hanya saat mengklik **pilih sekali klik**, skrip akan memproses sesuai aturan: menyimpan 1 item di setiap grup dan mencentang sisanya.

Selain itu, **pilih berdasarkan folder** di Cari Duplikat File akan memilih file dalam folder tertentu dari daftar duplikat; **balik pilihan** akan memilih file di luar folder tersebut dari daftar duplikat.

Perlu diperhatikan bahwa **Cocok Presisi** di Cari Duplikat File menggunakan kecocokan hash, biasanya cocok untuk pembersihan bersama pilih sekali klik. Sementara **Durasi Mirip / Nama Mirip** di Cari Duplikat File, serta duplikat folder, adalah kecocokan algoritma kemiripan. Disarankan memeriksa manual sebelum menghapus agar tidak salah hapus.

**Q: Mengapa "Cocok Presisi" di skrip mengatakan tidak ada file duplikat, tetapi klien menampilkan file duplikat? Setelah refresh atau ganti browser tetap tidak terpindai, apa yang harus dilakukan?**  
**A:** "Cocok Presisi" di skrip menilai berdasarkan **hash konten file**, bukan nama file. Hanya saat hash file dan ukuran file sama, file dianggap duplikat presisi.

"File duplikat" yang ditampilkan klien tidak selalu berarti file dengan konten benar-benar sama. Itu bisa berupa file bernama sama, file dengan akhiran `(1)` / `(2)`, atau file dengan ukuran dan durasi yang mirip. Selama konten sebenarnya berbeda, "Cocok Presisi" di skrip tidak akan mengelompokkannya sebagai duplikat.

Jika ingin mencari file yang "terlihat duplikat", gunakan mode **Nama Mirip** di Cari Duplikat File atau duplikat folder.

**Q: Mengapa beberapa video menampilkan pesan tidak dapat diputar saat pertama dibuka atau saat mengganti kualitas, tetapi bisa diputar lagi beberapa saat kemudian?**  
**A:** Situasi ini biasanya terkait status persiapan stream video resmi PikPak atau status decoding browser, dan tidak selalu merupakan kesalahan penilaian skrip.

Sebagian video di web resmi juga mungkin gagal dibuka pertama kali, gagal sebentar saat mengganti kualitas, lalu dapat diputar setelah dibuka ulang. Error tingkat bawah yang diterima skrip biasanya mirip:

```text
[VideoError] Code: 4, Msg: PipelineStatus::DEMUXER_ERROR_COULD_NOT_OPEN: FFmpegDemuxer: open context failed
```

Error ini tidak dapat membedakan secara stabil apakah "resource resmi masih disiapkan" atau "kualitas saat ini memang tidak dapat diputar". Jika skrip memaksanya sebagai "sedang disiapkan" dan terus mencoba ulang, video yang benar-benar tidak dapat diputar bisa salah dianggap dapat diputar, sehingga pengalaman menjadi lebih buruk.

Karena itu, skrip saat ini tetap menampilkan pesan "kualitas saat ini tidak tersedia" agar tidak menyesatkan pengguna.

Saat menemui situasi ini, Anda dapat mencoba:

* Tunggu beberapa detik lalu buka video lagi;
* Beralih ke kualitas lain;
* Coba putar dengan pemutar eksternal.

Jika nanti ditemukan field status resmi yang lebih andal, masalah salah peringatan ini dapat terus dioptimalkan.

**Q: Saat Ekstrak Massal muncul "sistem sibuk / menunggu coba ulang", apakah ini bug skrip?**  
**A:** Bukan. Ini biasanya karena server cloud PikPak memicu batas frekuensi konkuren. Artinya API resmi sedang sibuk, bukan skrip yang abnormal.

Jika suatu file tetap gagal setelah beberapa kali coba ulang, biasanya resource tersebut di cloud mungkin sudah rusak, dibatasi, atau kondisi server sedang buruk. Disarankan mencoba lagi nanti, atau unduh file ke lokal lalu tangani.

**Q: Setelah hapus massal lalu restore, jumlah file benar tetapi file tidak kembali ke folder semula dan struktur direktori tampak kacau. Bisakah dipulihkan sekali klik?**  
**A:** Situasi ini biasanya bukan kesalahan operasi dan bukan logika skrip yang menghilangkan file. Ini lebih mirip perilaku recycle bin resmi PikPak saat restore dilakukan dari **jalur virtual / hasil agregasi**.

Saat file dihapus massal dari **tampilan agregasi atau jalur virtual** seperti Cari Duplikat File, Tinjau File, tinjau folder, terbaru ditambahkan, atau favorit, file-file tersebut mungkin berasal dari banyak folder asli yang berbeda. Setelah dihapus lalu dipulihkan dari recycle bin, mekanisme restore resmi terkadang tidak mengembalikan satu per satu ke folder induk asalnya, tetapi dapat memulihkan banyak file ke satu direktori terpadu sehingga struktur bertingkat asli rusak.

Sebelum menghapus massal dari hasil agregasi seperti Cari Duplikat File, Tinjau File, tinjau folder, terbaru ditambahkan, atau favorit, pastikan file tersebut memang tidak diperlukan lagi. Terutama jika file berasal dari banyak direktori berbeda, sebisa mungkin hindari mengandalkan restore massal recycle bin setelah penghapusan.

**Q: Mengapa saat menghapus file secara massal, sebagian file ditandai terlindungi atau tidak dapat dihapus?**  
**A:** Periksa terlebih dahulu apakah file tersebut sudah tercatat di **Pengelola Sumber Daya**. Jika aturan "lewati resource yang tercatat di pengelola saat menghapus" aktif di Pengaturan, skrip akan menganggap file yang cocok sebagai item terlindungi untuk mencegah salah hapus.

**Solusi:**

1. Buka Pengaturan dan hapus centang aturan perlindungan tersebut, lalu jalankan penghapusan lagi;
2. Atau klik entri **"Pengelola Sumber Daya"** di toolbar / sidebar, pilih **"Jalankan Pembersihan Sekarang"**, dan lakukan pembersihan fisik paksa pada item tercatat tersebut.

**Q: Mengapa setelah menjalankan operasi "tempel", file tidak muncul di daftar?**  
**A:** Periksa dulu apakah ruang tersisa di cloud drive Anda cukup. PikPak resmi saat ini biasanya memakai strategi **blokir diam-diam** untuk "ruang tidak cukup", yaitu saat kapasitas terlampaui, belum tentu ada notifikasi jelas dan operasi bisa dihentikan langsung di latar belakang.

Jadi jika file tidak muncul setelah tempel, penyebab paling umum adalah batas kapasitas. Bersihkan ruang terlebih dahulu, lalu coba lagi.

**Q: Apa itu "Migrasi Data multi-akun" dan bagaimana cara menggunakannya?**  
**A:** Fitur ini dapat mentransfer file atau folder dari akun saat ini ke akun PikPak Anda yang lain dengan cepat.

**Langkah:**

1. Pilih file atau folder yang akan dimigrasikan di akun saat ini;
2. Klik tombol **"Migrasi Data"** di bagian bawah;
3. Skrip akan otomatis mengenkripsi dan mengemas data, lalu keluar dari akun saat ini;
4. Setelah itu, cukup masuk normal ke akun tujuan;
5. Setelah login berhasil, skrip akan otomatis mendeteksi paket migrasi di cache lokal dan menampilkan popup untuk menerima sekali klik.

## Pernyataan Privasi dan Keamanan

* **Utamakan lokal**: semua kemampuan inti skrip ini berinteraksi langsung dengan API resmi PikPak melalui browser. Token akun, Brankas Kata Sandi Ekstraksi, dan sebagian besar data konfigurasi lokal Anda secara default disimpan di lingkungan browser lokal.
* **Tanpa pengumpulan**: skrip **tidak secara aktif mengumpulkan** data privasi pengguna, dan **tidak akan pernah** mengunggah informasi file atau kredensial akun Anda ke server pihak ketiga mana pun.
* **Antarmuka pihak ketiga**: hanya saat menggunakan fitur tambahan seperti "pencarian subtitle online", "pencarian gambar", atau "pratinjau magnet", skrip akan mengirim kata kunci pencarian, parameter fitur gambar, atau informasi fitur magnet yang diperlukan ke layanan publik terkait. Ini tidak melibatkan informasi identitas pribadi Anda.
* **Batas hak membership**: skrip ini hanya untuk meningkatkan pengalaman penggunaan PikPak Web. Skrip ini **tidak memiliki kemampuan membobol, melewati, atau memalsukan hak membership PikPak**, dan tidak menyediakan fitur unlock membership apa pun yang melanggar aturan layanan resmi.

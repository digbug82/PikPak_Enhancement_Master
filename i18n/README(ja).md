## 主な機能

### 体験とナビゲーション強化

* **サイドボタンナビゲーション**：マウスのサイドボタンによる進む / 戻るをサポートし、異なるフォルダ階層をすばやく切り替えられます。
* **高度なパスバー**：ホイールスクロール、同階層フォルダのドロップダウン切り替え、パス表示、元パスへのトレースジャンプをサポートします。
* **全体検索強化**：ファイル名とファイルパスで検索でき、キーワード部分を赤く強調し、長いファイル名ではヒットしたキーワード周辺を優先表示します。
* **閲覧設定**：並び替え設定、表示設定、閲覧位置の保持、メディアモード、ボタン文字の非表示、メディアカバーサムネイルのぼかし、ダークモードをサポートします。

### リンクブックマークとクラウドアーカイブ

* **リンクブックマーク**：リンクブックマークは、ブックマークリンク、スクリプト同期データ、クラウドアーカイブのマグネットリンク、クラウド設定を同時に保存します。
* **クラウドアーカイブ**：選択したファイル内の追跡可能なマグネットリンクを「リンクブックマーク > クラウドアーカイブ」にアーカイブでき、元ファイルを削除して容量を節約できます。
* **クラウドアーカイブ復元解析**：アーカイブ済みのマグネットリンクをクラウドダウンロードタスクとして再送信し、送信状態を書き戻せます。
* **クラウドアーカイブ整理**：すべてのクラウドアーカイブリンクのクリア、送信済みリンクの削除、個別アーカイブリンクの削除に対応します。
* **設定クラウド同期**：スクリプト設定のクラウドへのアップロード、クラウドからの取得、クラウド設定のクリアに対応し、複数デバイス間でスクリプト設定を同期しやすくします。

### 転送、クラウドダウンロード、共有解析

* **共有解析**：共有リンクを解析し、保存前に共有内容を確認できます。保存するまでは自分のクラウド容量を消費しません。
* **共有ファイル透視**：共有内容の再帰スキャン、フィルタ、プレビューをサポートし、保存前に共有内部の構造を確認するのに適しています。
* **共有解析履歴**：解析済みの共有リンクとパスワードを記録し、後からすばやく開き直せます。
* **共有管理**：共有の取得回数上限を設定できます。条件に達すると自動で共有を取り消し、リンクを無効化できます。
* **秒アップロード**：アップロード時にサーバー側に同一ファイルの記録がある場合、即時にアップロードを完了でき、重複アップロードの待ち時間を短縮します。
* **アップロード保護**：アップロード中のページ離脱通知を内蔵し、アップロード中断、残留タスク、異常ファイルの保護と整理を行います。
* **クラウドダウンロード強化**：一括オフラインリンクの自動重複排除をサポートします。内蔵のスマートマグネットクリーニングにより、Base32 / Hex ハッシュを抽出し、ノイズテキストを除去できます。
* **TXT マグネット抽出**：TXT テキストをプレビューし、テキスト内のマグネット、HTTP、FTP、ed2k、thunder などのダウンロードリンクを識別してクラウドダウンロードへ送信できます。
* **マグネットプレビュー**：マグネット内容に公開プレビュー情報がある場合、ファイル数、合計サイズ、種類、ハッシュ、複数のプレビュー画像を表示できます。

* **再生エンジン強化**：0.5x - 3.0x 倍速、回転 / 反転、強制アスペクト比、オープニング / エンディングの自動スキップ、連続再生 / ループ、進捗バーのサムネイルプレビューをサポートします。
* **デフォルトの開き方**：スクリプトプレイヤーと PotPlayer の間でデフォルト再生方式を選択でき、デフォルト動画品質 / 外部再生品質設定にも対応します。
* **音楽プレイヤー**：音声ファイルのオンライン再生、プレイリスト、順番再生、シャッフル、1 曲ループ、ポップアップモード、カバー表示、音量制御をサポートします。
* **字幕システム**：クラウド上の同名字幕読み込み、ローカル字幕インポート、オンライン字幕検索をサポートします。字幕オフセット、位置、フォントサイズ、背景透明度の調整にも対応します。

## よくある質問

**Q：スクリプトをインストールしたのに青いフローティングボールが表示されないのはなぜですか？**  
**A：** 次の順番で確認してください。

1. ユーザースクリプトマネージャーで **PikPak Enhancement Master** が有効になっていることを確認します。
2. PikPak Web アカウントにログイン済みであることを確認します。
3. **Tampermonkey** を使用している場合は、ブラウザ拡張機能の詳細ページで **「ユーザースクリプトを許可」** がオンになっていることを確認します。
4. **Edge / Chrome** の最新版ブラウザの使用をおすすめします。
5. 上記を完了してもフローティングボールが表示されない場合は、他のブラウザ拡張機能を一時的にすべて無効化し、ページを更新（F5）して再試行してください。

**Q：深い階層のフォルダから戻るのが面倒です。メイン画面から入り直すしかありませんか？**  
**A：** 必要ありません。スクリプト内のパスバーがあるすべてのページは、**マウスホイールの横スクロール**に対応しています。

ディレクトリ階層が深く、前方の上位フォルダパスが隠れている場合は、マウスを上部のパスバー領域に移動し、マウスホイールをスクロールします。パスバーが横にスクロールし、隠れていた上位パスが表示されます。

表示されたら、対応する上位フォルダ名を直接クリックすることで、すばやく一つ上の階層に戻るか、任意の上位ディレクトリへ移動できます。

**Q：ファイル重複チェック / フォルダ重複チェックで選択されたファイルは元ファイルを 1 つ自動で残しますか？それともリストで再度絞り込む必要がありますか？**  
**A：** ファイル重複チェックとフォルダ重複チェックの選択項目は、本質的に **実際の操作対象** です。重複チェックモード自体はファイルを直接削除せず、重複の可能性がある項目を一覧表示するだけです。確認して選択した後、削除をクリックして初めて実際に整理されます。

重複チェック結果では：

* 各グループ見出しのチェックボックスは、そのグループ全体を選択することを意味します；
* グループ内並び替えは、時間、サイズ、パス、名前などの規則で表示順を並べ替えるだけで、誰を削除し誰を残すかを自動決定しません；
* **ワンクリック選択** をクリックしたときだけ、スクリプトが規則に従って各グループ 1 件を残し、残りの項目を選択します。

また、ファイル重複チェックの **フォルダ別選択** は、重複リスト内の特定フォルダ内ファイルを選択します。**反転選択** は、そのフォルダ以外のファイルを選択します。

注意点として、ファイル重複チェックの **完全一致** はハッシュ値マッチングを使用するため、通常はワンクリック選択と組み合わせた整理に適しています。一方、ファイル重複チェックの **長さ類似 / 名前類似** およびフォルダ重複チェックは類似アルゴリズムによる判定なので、誤削除を避けるため削除前の手動確認をおすすめします。

**Q：スクリプトの「完全一致」では重複ファイルがないと表示されますが、クライアントでは重複ファイルが見えます。ページ更新やブラウザ変更でもスキャンできない場合はどうすればよいですか？**  
**A：** スクリプトの「完全一致」は **ファイル内容ハッシュ** で判断しており、ファイル名では判断しません。ファイルのハッシュ値とサイズがどちらも一致した場合のみ、完全重複と判定されます。

クライアントに表示される「重複ファイル」は、必ずしも内容が完全に同じファイルとは限りません。同名ファイル、ファイル名末尾に `(1)` / `(2)` が付いたファイル、サイズや再生時間が近いファイルが含まれることがあります。実際の内容に差異がある限り、スクリプトの「完全一致」はそれらを重複として扱いません。

このような「見た目が重複している」ファイルを探したい場合は、ファイル重複チェックまたはフォルダ重複チェックの **名前類似** モードを使用できます。

**Q：一部の動画は初回再生や画質切り替え時に再生不可と表示されますが、しばらくして再度開くと再生できるのはなぜですか？**  
**A：** このような状況は通常、PikPak 公式動画ストリームの準備状態やブラウザのデコード状態に関係しており、必ずしもスクリプト側の判断ミスではありません。

一部の動画は公式 Web 端でも、初回に開けない、画質切り替え直後に短時間失敗する、しばらくして再度開くと再生できる、といったことがあります。スクリプトが受け取る低レベルエラーは一般的に次のようなものです。

```text
[VideoError] Code: 4, Msg: PipelineStatus::DEMUXER_ERROR_COULD_NOT_OPEN: FFmpegDemuxer: open context failed
```

このエラーでは、「公式リソースがまだ準備中」なのか、「現在の画質が本当に再生不可」なのかを安定して区別できません。スクリプトが強制的に「準備中」とみなして再試行し続けると、本当に再生できない動画まで再生可能と誤判定し、より悪い体験につながる可能性があります。

そのため、現在のスクリプトではユーザーを誤解させないよう、「現在の画質は使用できません」という案内を保持しています。

この場合は、次を試してください。

* 数秒待ってから動画を再度開く；
* 別の画質へ切り替える；
* 外部再生を試す。

今後、公式のより信頼できる状態フィールドが見つかれば、この種の誤検出をさらに最適化できます。

**Q：一括解凍時に「システムが混雑しています / 再試行待ち」と表示されます。スクリプトのバグですか？**  
**A：** いいえ。これは通常、PikPak クラウドサーバーが同時実行頻度制限を発動したためで、公式 API が現在混雑している状態であり、スクリプト自体の異常ではありません。

あるファイルが複数回再試行しても失敗する場合、そのリソースがクラウド上で破損、制限、または現在のサーバー状態不良の影響を受けている可能性があります。しばらくしてから再試行するか、ファイルをローカルにダウンロードして処理することをおすすめします。

**Q：一括削除後に復元すると、ファイル数は正常なのに元のフォルダへ戻らず、ディレクトリ構造が乱れたように見えます。ワンクリックで復元できますか？**  
**A：** この状況は通常、操作ミスでもスクリプトがファイルを失ったわけでもなく、PikPak 公式ごみ箱が **仮想パス / 集約結果** で復元を実行したときの挙動に近いものです。

ファイル重複チェック、ファイル透視、フォルダ透視、最近追加、スター付きなどの **集約ビューまたは仮想パス** でファイルを一括削除した場合、それらのファイルはもともと複数の異なる元フォルダに存在していた可能性があります。削除後にごみ箱から復元すると、公式の復元機構が各ファイルを元の親ディレクトリへ個別に戻さず、多数のファイルを一つの統一ディレクトリへまとめて復元し、元の多層ディレクトリ構造が壊れる場合があります。

ファイル重複チェック、ファイル透視、フォルダ透視、最近追加、スター付きなどの集約結果で一括削除する前に、それらのファイルが本当に不要か確認してください。特にファイルが複数の異なるディレクトリから来ている場合、削除後にごみ箱の一括復元へ依存することは避けるのが無難です。

**Q：ファイルを一括削除するとき、一部のファイルが保護されている、または削除できないと表示されるのはなぜですか？**  
**A：** まず、これらのファイルが **リソース管理** に記録されていないか確認してください。設定で「削除時に管理器に記録されたリソースをスキップ」を有効にしている場合、スクリプトは一致したファイルを保護対象とみなし、誤削除を避けます。

**解決方法：**

1. 設定でこの保護ルールのチェックを外してから、再度削除を実行します；
2. またはツールバー / サイドバーの **「リソース管理」** 入口をクリックし、**「今すぐ整理を実行」** を選択して、これらの記録項目に強制的な物理整理を行います。

**Q：「貼り付け」操作を実行した後、ファイルがリストに表示されないのはなぜですか？**  
**A：** まず、クラウドドライブの空き容量が十分か確認してください。PikPak 公式は現在、「容量不足」に対して通常 **サイレントブロック** を採用しており、容量超過時に明確な通知を出さず、バックグラウンドで操作を中止することがあります。

そのため、貼り付け後にファイルが表示されない場合、最も一般的な原因は容量制限です。先に容量を整理してから再試行してください。

**Q：「複数アカウントデータ移行」とは何ですか？具体的にどう使いますか？**  
**A：** この機能は、現在のアカウント内のファイルまたはフォルダを、別の PikPak アカウントへすばやく転送できます。

**操作手順：**

1. 現在のアカウントで移行したいファイルまたはフォルダを選択します；
2. 下部の **「データ移行」** ボタンをクリックします；
3. スクリプトが自動的にデータを暗号化してパッケージ化し、現在のアカウントからログアウトします；
4. その後、通常どおり対象アカウントにログインします；
5. ログイン成功後、スクリプトがローカルキャッシュ内の移行パッケージを自動検出し、ワンクリック受け取りのポップアップを表示します。

## プライバシーとセキュリティ声明

* **ローカル優先**：本スクリプトのすべての主要機能は、ブラウザを通じて PikPak 公式 API と直接通信します。アカウント Token、解凍パスワードライブラリ、および大部分のローカル設定データは、既定でローカルのブラウザ環境に保存されます。
* **収集ゼロ**：スクリプトはユーザーのプライバシーデータを **能動的に収集せず**、ファイル情報やアカウント認証情報を **決して** 第三者サーバーへアップロードしません。
* **第三者インターフェース**：「オンライン字幕検索」「画像検索」「マグネットプレビュー」などの拡張機能を使用する場合のみ、必要な検索キーワード、画像特徴パラメータ、またはマグネット特徴情報を関連する公開サービスへ送信します。個人識別情報は含まれません。
* **会員権益の境界**：このスクリプトは PikPak Web 端の操作体験を強化するためのものであり、**PikPak 会員権益をクラック、回避、または偽造する能力はありません**。また、公式サービス規則に違反する会員解放機能も提供しません。

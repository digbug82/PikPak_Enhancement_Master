## ✨ 主な機能

### ✨ 体験とナビゲーションエンジン

* **サイドボタンナビゲーション**：マウスのサイドボタンで進む / 戻るを行い、異なるディレクトリ階層をすばやく切り替えられます。
* **高度なパスバー**：ホイールスクロール、同階層ディレクトリのドロップダウン切り替え、パス表示、遡ってジャンプする操作をサポートします。
* **全体検索の強化**：ファイル名とファイルパスで検索でき、キーワード一致部分を赤く表示し、長いファイル名では一致キーワード周辺を優先して表示します。
* **閲覧設定**：並び替え設定、表示設定、閲覧位置の保持、メディアモード、ボタン文字の非表示、メディアカバーサムネイルのぼかし、夜間モードをサポートします。

### 🌐 転送、クラウドダウンロード、共有解析

* **共有ファイル透視**：共有内容を再帰的にスキャンしてフィルタし、保存前に共有内部の構造をプレビューできます。
* **共有管理**：共有の抽出回数上限を設定できます。条件に達すると共有を自動的にキャンセルし、リンクを無効化できます。
* **アップロード保護**：アップロード中の離脱リマインダーを内蔵し、アップロード中断、残留タスク、異常ファイルを保護およびクリーンアップします。
* **クラウドダウンロード強化**：一括オフラインリンクの自動重複排除に対応します。内蔵のマグネットスマートクリーニングで Base32 / Hex ハッシュを抽出し、ノイズテキストを除去できます。
* **TXT マグネット抽出**：TXT テキストをプレビューし、テキスト内のマグネット、HTTP、FTP、ed2k、thunder などのダウンロードリンクを識別してクラウドダウンロードへ送信できます。
* **マグネットプレビュー**：マグネット内容に複数のプレビュー画像がある場合、複数画像プレビューに対応します。

### ⚡ ダウンロード、外部再生、配信

* **既定の開き方**：スクリプトプレイヤーと PotPlayer の間で既定の再生方法を選択できます。
* **PotPlayer プロトコル修復アシスタント**：PotPlayer プロトコルの検出と修復補助を内蔵し、ブラウザから外部プレイヤーを起動できない問題を減らします。
* **RPC 配信**：RPC プロトコルを通じて、ファイルを Aria2 / Motrix などのダウンロードノードへ送信できます。
* **ディレクトリ構造の復元**：フォルダ全体を送信する際、クラウドドライブ内のツリー構造を自動復元し、ダウンロード後にディレクトリが平坦化されるのを防ぎます。
* **ダウンロードフィルタ**：ファイルサイズ、拡張子、名前キーワードでファイルをフィルタし、ブラウザダウンロードと Aria2 / Motrix 送信に適用できます。
* **ダウンロード直リンク高速化ドメイン**：カスタムダウンロード高速化ドメインに対応し、接頭辞付き完全元リンクと URL パラメータ受け渡しの 2 種類のリバースプロキシ書き換えモードを提供します。
* **例外処理**：送信時に 0KB の無効ファイルを自動スキップし、失敗リストの出力に対応します。

## ❓ よくある質問

**Q：スクリプトをインストールしたのに青いフローティングボールが表示されないのはなぜですか？**  
**A：** 次の順序で確認してください：

1. スクリプトマネージャーで **PikPak Enhancement Master** が有効になっていることを確認します。
2. PikPak Web アカウントにログインしていることを確認します。
3. **Tampermonkey** を使用している場合は、ブラウザ拡張機能の詳細ページで **「ユーザースクリプトを許可」** がチェックされていることを確認します。
4. 最新版の **Edge / Chrome** ブラウザの使用をおすすめします。
5. 上記の手順を完了してもフローティングボールが表示されない場合は、他のすべてのブラウザ拡張機能を一時的に無効化し、ページを更新（F5）して再試行してください。

**Q：多階層フォルダで 1 つ上の階層に戻るのが面倒です。メイン画面から入り直すしかありませんか？**  
**A：** いいえ。スクリプト内でパスバーがあるすべてのページは、**マウスホイールの横スクロール**に対応しています。

ディレクトリ階層が深く、前方の上位フォルダパスが隠れている場合は、マウスを上部のパスバー領域に移動し、マウスホイールをスクロールします。パスバーが横方向にスクロールし、隠れていた上位パスが表示されます。

表示後、該当する上位フォルダ名を直接クリックすれば、すばやく 1 つ上の階層へ戻る、または任意の上位ディレクトリへジャンプできます。

**Q：重複チェック / フォルダ重複チェック時、チェックされたファイルは元ファイルを 1 つ自動で残しますか？それともリストで再度絞り込む必要がありますか？**  
**A：** 重複チェックとフォルダ重複チェックのチェック項目は、本質的には**実際の操作対象**です。重複チェックモード自体はファイルを直接削除せず、重複の可能性がある項目を一覧表示するだけです。確認してチェックしたあと、削除をクリックして初めて実際にクリーンアップされます。

重複チェック結果では：

* 各グループ見出しのチェックボックスは、そのグループ全体を選択することを意味します；
* グループ内ソートは時間、サイズ、パス、名前などのルールに従って表示順を並べ替えるだけで、何を削除し何を残すかを自動決定しません；
* **一括選択**をクリックした場合のみ、スクリプトがルールに従って自動処理します。各グループで 1 件を残し、残りの項目がチェックされます。

また、重複チェックの **フォルダごとに選択** は、重複リスト内の特定フォルダ内のファイルをチェックします。**選択反転** は、そのフォルダ以外の重複リスト内ファイルをチェックします。

注意点として、重複チェックの **完全一致** はハッシュ値による照合を使用するため、通常は一括選択と組み合わせたクリーンアップに適しています。一方で、重複チェックの **長さ類似 / 名前類似** とフォルダ重複チェックは、いずれも類似アルゴリズムによる照合なので、誤削除を避けるため削除前の手動確認をおすすめします。

**Q：スクリプトの「完全一致」では重複ファイルがないと表示されるのに、クライアントでは重複ファイルが見えます。ページ更新やブラウザ変更でもスキャンできない場合はどうすればよいですか？**  
**A：** スクリプトの「完全一致」は**ファイル内容のハッシュ**で判定しており、ファイル名では判定していません。ファイルのハッシュ値とファイルサイズがどちらも一致した場合にのみ、完全な重複として判定されます。

クライアントが表示する「重複ファイル」は、必ずしも内容が完全に同一のファイルとは限りません。同名ファイル、ファイル名末尾に `(1)` / `(2)` が付いたファイル、またはサイズや長さが近いファイルを含む場合があります。実際の内容に差異があれば、スクリプトの「完全一致」はそれらを重複として分類しません。

このような「重複に見える」ファイルを探したい場合は、重複チェックまたはフォルダ重複チェックの **名前類似** モードを使用できます。

**Q：一部の動画で、初回再生時または画質切り替え時に再生不可と表示されるのに、少し後で開くと再生できるのはなぜですか？**  
**A：** このような状況は通常、PikPak 公式動画ストリームの準備状態やブラウザのデコード状態に関係しており、必ずしもスクリプト側の判定ミスとは限りません。

一部の動画は公式 Web 版でも、初回に開けない、画質切り替え直後に短時間失敗する、少し後に開き直すと再生できる、といった現象が発生する場合があります。スクリプトが受け取る低レベルのエラーは一般的に次のようなものです：

```text
[VideoError] Code: 4, Msg: PipelineStatus::DEMUXER_ERROR_COULD_NOT_OPEN: FFmpegDemuxer: open context failed
```

このエラーは「公式リソースがまだ準備中」なのか、「現在の画質が本当に再生不可」なのかを安定して区別できません。スクリプトが無理に「準備中」と見なして再試行を続けると、本当に再生できない動画まで再生可能と誤判定して、より悪い体験につながる可能性があります。

そのため、スクリプトは現在「現在の画質は利用できません」という案内を残し、ユーザーを誤導しないようにしています。

この状況に遭遇した場合は、次を試してください：

* 数秒待ってから動画を開き直す；
* 別の画質に切り替える；
* 外部再生で再生を試す。

今後、公式のより信頼できる状態フィールドが見つかれば、この種の誤検知をさらに最適化できます。

**Q：一括解凍時に「システムビジー / 再試行待ち」と表示されます。スクリプトの Bug ですか？**  
**A：** いいえ。これは通常、PikPak クラウドサーバー側で同時実行または頻度制限が発生したためで、公式インターフェースが現在ビジーであることを意味します。スクリプト自体の異常ではありません。

あるファイルが複数回再試行しても失敗する場合、そのリソースがクラウド上で破損している、制限されている、または現在のサーバー状態がよくない可能性があります。時間を置いて再試行するか、ファイルをローカルにダウンロードしてから処理することをおすすめします。

**Q：一括削除後に復元すると、ファイル数は正常ですが元のフォルダに戻らず、ディレクトリ構造が乱れたように見えます。ワンクリックで復元できますか？**  
**A：** この状況は通常、操作ミスでもスクリプトロジックによるファイル紛失でもありません。PikPak 公式のごみ箱が **仮想パス / 集約結果** から復元を実行したときの挙動に近いものです。

ファイルが重複チェック、ファイル透視、フォルダ透視、最近の追加、スター付きなどの **集約ビューまたは仮想パス** で一括削除された場合、それらのファイルは複数の異なる元フォルダから来ている可能性があります。削除後にごみ箱から復元すると、公式の復元メカニズムが各項目を元の親ディレクトリへ個別に戻さず、大量のファイルを 1 つの統一ディレクトリへまとめて復元することがあり、元の多階層ディレクトリ構造が壊れる場合があります。

重複チェック、ファイル透視、フォルダ透視、最近の追加、スター付きなどの集約結果で一括削除する前に、それらのファイルが本当に不要かどうかを確認してください。特に複数の異なるディレクトリから来たファイルの場合、削除後にごみ箱の一括復元へ依存することはできるだけ避けてください。

**Q：ファイルを一括削除するとき、一部のファイルが保護されている、または削除できないと表示されるのはなぜですか？**  
**A：** まず、それらのファイルが **リソース管理** に記録されているか確認してください。設定で「削除時に記録済みリソースをスキップ」を有効にしている場合、スクリプトは一致したファイルを保護対象と見なし、誤削除を防ぎます。

**解決方法：**

1. 設定でこの保護ルールのチェックを外し、削除を再実行します；
2. またはツールバー / サイドバーの **「リソース管理」** 入口をクリックし、**「今すぐクリーンアップ」** を選択して、これらの記録項目に強制物理クリーンアップを実行します。

**Q：「貼り付け」操作を実行したあと、ファイルがリストに表示されないのはなぜですか？**  
**A：** まず、クラウドドライブの空き容量が十分か確認してください。PikPak 公式は現在「容量不足」に対して通常 **サイレントブロック** 戦略を採用しています。つまり容量超過時に明確な案内が表示されず、バックグラウンドで操作が直接中止される場合があります。

そのため、貼り付け後にファイルが表示されない最も一般的な原因は、容量制限に引っかかったことです。先に容量を整理してから再試行してください。

**Q：「複数アカウントデータ移行」とは何ですか？具体的にはどう操作しますか？**  
**A：** この機能は、現在のアカウント内のファイルまたはフォルダを、あなたの別の PikPak アカウントへすばやく転送できます。

**操作手順は次のとおりです：**

1. 現在のアカウントで移行したいファイルまたはフォルダを選択します；
2. 下部の **「データ移行」** ボタンをクリックします；
3. スクリプトがデータを自動で暗号化してパッケージ化し、現在のアカウントから自動ログアウトします；
4. その後、対象アカウントへ通常どおりログインするだけです；
5. ログイン成功後、スクリプトはローカルキャッシュ内の移行パッケージを自動検出し、ワンクリック受信を促すポップアップを表示します。

## 🛡️ プライバシーと安全声明

* **ローカル優先**：このスクリプトのすべてのコア機能は、ブラウザを通じて PikPak 公式 API と直接通信します。アカウント Token、パスワード庫、大多数のローカル設定データは既定でローカルブラウザ環境に保存されます。
* **収集ゼロ**：スクリプトはユーザーのプライバシーデータを**能動的に収集せず**、あなたのファイル情報やアカウント資格情報をいかなる第三者サーバーにも**絶対にアップロードしません**。
* **第三者インターフェース**：「オンライン字幕検索」や「画像で検索」などの拡張機能を使用する場合のみ、必要な検索キーワードまたは画像特徴パラメータを関連する公共サービスへ送信します。個人を特定する情報は含まれません。

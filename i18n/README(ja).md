## ✨ 主な機能 (Features)

### ✨ エクスペリエンス＆ナビゲーションエンジン

* **インタラクションの再構築**：インターフェースを Windows エクスプローラー風に設計し、操作体験を全面的に向上させました。
* **極速モード**：Web版のネイティブロジックを代替し、標準の同期処理をブロックします（**設定での有効化を推奨**）。
* **高機能パスバー**：マウスホイールスクロール、ドロップダウン切り替え、パスの履歴表示と遡及ジャンプに対応しています。
* **バックグラウンド自動インデックス**：ホームアイコンの青い点滅でディレクトリツリーの同期状態を通知します。
* **UXの強化**：ダークテーマ、動画の長さ表示、ワンクリックでのカバー画像をぼかす（プライバシーモード）、スター付き、ファイル種別ソートをサポート。

### 🌐 転送＆共有センター

* **共有管理**：抽出回数の上限設定に対応し、回数に達すると自動的に共有を解除します。
* **オフライン転送**：一括オフラインタスク＋リンク出力。
* **高速アップロード**：超大容量ファイルのアップロードに対応し、小容量ファイルのアップロード失敗率を大幅に低減。
* **クラウドDL拡張**：一括リンク追加時の**自動重複排除**をサポート。内蔵の**マグネットスマートクリーンエンジン**により、文字ノイズから磁気リンクを自動抽出し、Base32デコードに対応。

> 2. **スクリプトのインストール**: **[今すぐインストール](https://github.com/digbug82/PikPak_Enhancement_Master/raw/main/PikPak_Enhancement_Master.user.js)** をクリック。

> 4. **スクリプト有効化**: Webにログイン後、サイドバーに表示される青い **PikPak ロゴの球体** をクリックして拡張マスターを起動します。未ログイン状態では起動できません。

## ❓ よくある質問 (FAQ)

**Q: スクリプトをインストールしたのに青い浮遊球体が表示されません。**  
**A:** PikPak Web版にログインしているか確認してください。ログインしても表示されない場合は、ページをリロード (F5) してみてください。または、広告ブロックプラグイン（AdGuard / uBlock Origin など）に誤ってブロックされていないか確認し、プラグインを一時的に無効化するか、このサイトをホワイトリストに追加してください。

**Q: 「極速モード (Turbo Mode)」とは何ですか？なぜ有効化が推奨されるのですか？**  
**A:** 有効にすると、スクリプトがWeb UIを全画面で覆い、標準のWeb機能を完全に制御します。このモードはメモリを大量に消費するネイティブの同期ロジックを物理的にブロックすることで、大量のファイルを扱う際のフリーズやクラッシュを根本的に解決します。**注意:** 極速モード中は最小化や閉じるボタンが自動的に隠されます（元のWebに戻るには設定でチェックを外してください）。**推奨:** ファイル数が多い場合、究極のレスポンス速度とスムーズな体験を得るために、このモードの有効化を強く推奨します。

**Q: ホームの「My Pack」フォルダに「既定」と表示され、スター追加、コピー、移動、リネーム、削除ができないのはなぜですか？**  
**A:** これは公式のセキュリティ保護メカニズムです。スクリプトはこれに基づき、デフォルトフォルダである My Pack の共有制限のみを解除しています。

**Q: スクリプト内の「ファイル重複チェック (ファイル分析)」と「フォルダ重複チェック (フォルダ分析)」の根本的な違いは何ですか？**  
**A:** **ファイル重複チェック:** 「単一ファイル」レベルを対象とします。完全一致（ハッシュ）、長さ類似、名前類似の3つのアルゴリズムを使用し、クラウド内に散在する重複ファイル（重複保存された単一エピソードや画像など）の整理に適しています。<br>
**フォルダ重複チェック:** 「フォルダの全体構造」レベルを対象とします。名前一致、内部ファイルの重複度（類似度一致）、サブセット冗長（包含率一致）のアルゴリズムを使用します。フォルダの「内部の真の姿」を見抜くため、完全に異なる名前のフォルダであっても、内部ファイルが高度に一致しているか包含関係があれば、重複保存された「シリーズ全体」や「画像パック全体」を正確に見つけ出すのに非常に適しています。

> **推奨:** ファイル重複チェックの「完全一致」はハッシュ照合のため、一括選択して削除しても安全です。ファイル重複チェックの長さ・名前類似、およびフォルダ重複チェックの各モードは類似度マッチングであるため、誤削除を防ぐために一括選択する前に必ず目視で確認してください。

**Q: 重複チェックで見つかる重複ファイルがだんだん増えていくのはなぜですか？全検索インデックスのデータが欠落しているのですか？**  
**A:** ご安心ください。データ欠落やエラーではなく、「完全一致」の検出数は常に安定しています。結果が増加するのは、動画の長さに基づく「類似一致」がより完全なものになっていくためです。PikPakの元データAPIは必ずしも動画の長さを直接提供しないため、初期段階ではデータ不足により多くの重複動画がスキップされます。これを解決するため、スクリプトにはバックグラウンドの非同期スニッフィングアルゴリズムが組み込まれており、動画メタデータを自動的に読み取って正確な長さを抽出し、ブラウザのローカルに永続的に保存します。使用期間が長くなるにつれてローカルの長さデータベースが充実し、エンジンは補完されたデータに基づいて、ファイル名が異なっても内容の長さが同じ隠れた重複リソースをより多く検出できるようになります。そのため、結果が徐々に増え、正確になっていきます。

**Q: 「画像で検索」の際、直接画像をアップロードできずペーストを求められることがあるのはなぜですか？**  
**A:** ブラウザのセキュリティポリシー（CORS クロスドメイン制限）によるものです。スクリプトがクラウドの画像を検索サーバーに直接プッシュできない場合、現在の画面のスクリーンショットを自動的にクリップボードにコピーします。開いた検索ページで `[Ctrl+V]` を押すだけで認識が完了します。

**Q: スクリプトで動画を再生する際、音声だけで映像が出ないのはなぜですか？**  
**A:** 多くの場合、その動画が m3u8 マルチオーディオ/マルチトラックストリーミングであり、クラウドでのトランスコード後に音声と映像のトラックが分離してしまったためです。Web版プレーヤーはまだこのような特殊なストリームのオンラインマルチトラック解析をサポートしていません。右下の「外部プレーヤーで再生」ボタンをクリックし、PotPlayer などのローカルの専用プレーヤーを呼び出してストリーミング再生することをお勧めします。

**Q: 画質メニューの「原画 (高速)」と通常の「原画」の違いは何ですか？**  
**A:** 両者の画質はほぼ完全に一致していますが、主な違いは背後のデコードおよび転送プロトコルにあります。「原画 (高速)」は公式がストリーミング用に最適化したリンクを使用しています。PotPlayerなどの外部プレーヤーを呼び出す際、「高速」と付くリンクを使用すると、最も高速な読み込みレスポンスとスムーズなシーク体験が得られます。Web再生でも外部ストリーミングでも、このオプションを優先して選択することをお勧めします。

**Q: スマホアプリや公式Web版で行った操作（アップロード、削除、移動など）が、スクリプトの分析リストにリアルタイムで同期されないのはなぜですか？**  
**A:** スクリプトは究極の検索・分析速度を保証するため、メモリスナップショット機能を採用しており、毎回数万件のファイル情報を再取得することはありません。解決策：他のクライアントでファイルを大量に変更した場合は、Webページをリロードして全検索インデックスを再取得してください。スクリプト操作中は複数デバイスでの同時データ変更を避けてください。

**Q: 一括解凍時、複数のファイルにそれぞれ異なるパスワードを一度に設定できますか？**  
**A:** 可能です。スマート照合ロジック：既知のパスワードをあらかじめ「パスワード庫」に保存しておけば、スクリプトは一括解凍時に各圧縮ファイルに対してパスワード庫内のすべてのパスワードを自動的に試行します。一致すれば、手動での操作なしに自動的に解凍を開始します。

**Q: 一括解凍時に「システム混雑中。リトライしています」と表示されるのはスクリプトのバグですか？**  
**A:** いいえ。これは通常、PikPak クラウドサーバーの同時処理頻度制限に達したことが原因です。このメッセージが表示される場合、クラウドの公式APIも現在混雑していることを意味します。複数回リトライしても失敗する場合は、そのリソースがクラウド上で破損しているか制限されている可能性があります。時間を置いてから再試行するか、ローカルにダウンロード・再アップロードして処理することをお勧めします。

**Q: 出力された JSON バックアップにはどのような情報が含まれますか？**  
**A:** JSON ファイルには、拡張マスター内で設定した重複チェックの好み、ブラック/ホワイトリストのルール、パスワード庫などのローカル設定データのみが含まれます。PikPak のアカウントやパスワードは**含まれていない**ため、クロスデバイスへの移行にも安心して使用できます。

**Q: バックアップを読み込むと、既存の設定やリストが上書きされますか？**  
**A:** いいえ。スクリプトは「スマート結合」ロジックにアップグレードされています。インポート時、リソース管理や履歴などのリスト項目はローカルデータと自動的に重複排除しながら結合されます。基本設定項目（検索エンジン、Aria2 アドレスなど）のみがインポートされたファイルに基づいて上書きされるため、ローカルに蓄積されたデータが失われることはありません。

**Q: ファイルを一括削除する際、一部のファイルが「保護されている」または削除できないと表示されるのはなぜですか？**  
**A:** それらのファイルが「リソース管理」に記録されていないか確認してください。設定で「削除時に記録済みリソースをスキップ」を有効にしている場合、スクリプトは一致するファイルを誤削除防止の保護対象として扱います。<br>
解決策: 
1. 設定でその保護ルールのチェックを外してから削除を行う。
2. ツールバー/サイドバーから「リソース管理」を開き、「今すぐクリーンアップ」をクリックして、記録された項目を強制的に物理削除する。

**Q: 「貼り付け」操作を実行したのに、ファイルがリストに表示されないのはなぜですか？**  
**A:** クラウドドライブの空き容量がこれらのファイルを保存するのに十分であるか確認してください。PikPak 公式は現在、「容量不足」に対して**サイレントブロック**（エラーポップアップを出さずにバックグラウンドで操作を中止する）を採用しています。貼り付け後にファイルが表示されない場合は、容量制限に達していることが多いため、空き容量を確保してから再試行してください。

**Q: クラウドDL作成画面の「規制回避磁気リンクの自動修復」とは何ですか？いつオンにするべきですか？**  
**A:** この機能は、リンクに混入している漢字、絵文字、記号などの文字ノイズを自動的に取り除き、特徴コード（ハッシュ）をインテリジェントに抽出して標準フォーマットに復元します。<br>
**推奨：この機能はデフォルトでチェックを入れておくことをお勧めします。** SNSから取得した不要な文字が大量に混ざったリンクや「先頭が欠けた」汚染されたマグネットリンクを自動的に処理できます。すでに `magnet:?` で始まる**標準的な磁気リンク**の場合は、自動的に修復処理をスキップし正常にダウンロードされます。

**Q: Aria2/Motrix やブラウザでフォルダをダウンロードする際、階層構造は維持されますか？**  
**A:** **Aria2/Motrix は階層を維持します**。スクリプトはディレクトリ階層のクローン技術を採用しており、ダウンロード完了後、ローカルハードドライブ上にクラウドのツリー構造が完璧に復元されます。また、Windows でサポートされていない特殊文字（`:` `*` `?` など）をフォルダ名から自動除去し、冗長なパスをスマートに短縮してダウンロードエラーを防ぎます。  
**ブラウザでのダウンロードは階層を維持しません**。ブラウザのネイティブダウンロードプロトコルの制限により、フォルダ内のすべてのファイルは同じディレクトリ内にフラットにダウンロードされます。**推奨:** 複雑なフォルダ構造を維持したい場合は、必ず「Aria2 へ送信」を優先して選択してください。

## 🛡️ プライバシー＆セキュリティ声明 (Privacy & Security)

* **データローカライゼーション**：本スクリプトのすべてのコア機能（一括リネーム、ファイル分析、解凍など）は、ブラウザを通じて直接 PikPak 公式 API と対話します。あなたのアカウントトークン、パスワード庫、およびファイルデータは**ローカルブラウザ内にのみ保存されます**。
* **収集ゼロ**：スクリプトはユーザーのプライバシーデータを一切収集**しません**。また、ファイル情報やアカウント資格情報を第三者のサーバーにアップロードすることは**絶対にありません**。
* **サードパーティAPI**：「オンライン字幕検索」や「画像で検索」などの拡張機能を使用する場合に限り、スクリプトは関連するパブリックサービスAPIに必要な検索キーワードや画像のパラメータを送信します。これに個人識別情報は一切含まれません。

## ✨ 主な機能 (Features)

### ✨ エクスペリエンスとナビゲーションエンジン

* **インタラクションの再構築**: 公式機能をベースに、インターフェースを**Windows エクスプローラー**風に再構築しました。
* **ターボモード**: 有効化するとネイティブロジックを引き継ぎ、大量のファイル環境で発生するラグやクラッシュの問題を完全に解決します。
* **高度なパスバー**: マウスホイールスクロールとドロップダウンメニューによる同レベルの切り替えと位置指定をサポートします。全体検索と分析スイートがパスバーに統合され、パスの表示とソース追跡のジャンプをサポートします。
* **エクスペリエンスの向上**: スター付けなどの多次元の並べ替え、およびワンクリックでの**ぼかしカバー**とダークモードの切り替えをサポートします。バックグラウンドでは**SWR戦略**を採用し、静かにシームレスにビューを更新します。
* **バックグラウンドインデックスと保護**: ホームページの青い点が点滅している場合は、ディレクトリツリーが同期中であることを示します。システム自体に同時操作の物理ロックがあり、競合する操作を傍受し、ダーティデータの生成を厳格に防ぎます。（注：デフォルトフォルダの My Pack は公式に保護されており、誤削除などの操作を厳格に防ぎます。）

### 🎬 没入型メディアの強化

* **再生エンジン**: 0.5倍～3.0倍速、回転/反転、アスペクト比の強制、オープニング/エンディングの自動スキップ、**連続/ループ**再生モードをサポートし、プログレスバーでのサムネイルプレビューをサポートします。組み込みの**ウォッチドッグ**は、黒い画面やサポートされていないエンコーディングに遭遇した場合、互換性のある画質に自動的にフォールバックします。

## ❓ よくある質問 (FAQ)

**Q: スクリプトをインストールしたのに、青いフローティングボールが表示されないのはなぜですか？**  
**A:** PikPak Webクライアントにログインしていることを確認してください。ログインしていても表示されない場合は、Webページを更新（F5）するか、AdGuard / uBlock Originなどの広告ブロック拡張機能によってブロックされていないか確認してください。一時的にブロック拡張機能を無効にするか、このサイトをホワイトリストに追加してみてください。

**Q: 「ターボモード (Turbo Mode)」とは何ですか？また、なぜ有効にすることが推奨されるのですか？**  
**A:** 有効にすると、スクリプトがWebページを全画面で覆い、すべてのネイティブWeb機能を完全に制御します。このモードは、元のWebページのメモリを大量に消費する同期ロジックを物理的にブロックすることにより、大量のファイルがある場合に発生するラグやクラッシュの問題を完全に解決します。**注意:** ターボモードでは、最小化および閉じるボタンが自動的に非表示になります（元のWebに戻るには、設定でチェックを外す必要があります）。**推奨:** ファイルが多い場合は、究極の応答速度とスムーズなエクスペリエンスを得るために、このモードを有効にすることを強くお勧めします。

**Q: ホームページの My Pack に「デフォルト」タグが表示され、スター付け、コピー、移動、名前変更、または削除ができないのはなぜですか？**  
**A:** これは公式のセキュリティ保護メカニズムです。スクリプトはこれに基づき、デフォルトフォルダ My Pack の共有制限のみを解除しています。

**Q: スクリプトの「ファイル重複排除」と「フォルダ重複排除」の根本的な違いは何ですか？実際の使用においてどのように選択すればよいですか？**  
**A:** **ファイル重複排除:** 「単一ファイル」レベルを対象とします。ハッシュの高精度マッチング、ビデオ再生時間の類似度、名前の類似度の3つのモードのアルゴリズムを使用し、クラウドドライブに散在する重複ファイル（例：重複して保存された単一のエピソード、単一の画像など）をクリーンアップするのに適しています。<br>
**フォルダ重複排除:** 「フォルダ全体の構造」レベルを対象とします。名前の一致、内部ファイルの重複度（類似度の一致）、サブセットの冗長性（包含率の一致）の3つのモードのアルゴリズムを使用します。フォルダの「内部の性質」を見抜くことができるため、重複して保存された「シリーズ全体」、「画像パック全体」、または「複数階層のデータ」をクリーンアップするのに非常に適しています。2つのフォルダの名前が完全に異なっていても、内部のファイルが高度に一致しているか、包含関係が存在する場合は、正確に見つけることができます。

> **推奨:** ファイル重複排除の高精度マッチングはハッシュの一致であるため、ワンクリックで選択して削除できます。ファイル重複排除の再生時間と名前の一致、およびフォルダ重複排除の3つのモードはすべて類似度の一致であるため、誤って削除しないように慎重に選択してください。

**Q: ファイル重複排除で検出される重複ファイルがどんどん増えているのはなぜですか？全体のインデックスでパケットロスが発生したのでしょうか？**  
**A:** ご安心ください。全体のインデックスのパケットロスやデータエラーではありません。重複排除の結果のうち、「高精度マッチング」部分の数は常に安定しています。重複排除の結果が増加している理由は、ビデオの再生時間に基づいた「類似度の一致」がより包括的になったためです。PikPakの一部の元のデータインターフェースはビデオの再生時間を直接提供しないため、初期段階ではデータ不足により多くの重複ビデオがスキップされていました。このため、スクリプトにはビデオメタデータの読み取りをシミュレートして正確な再生時間を抽出し、ブラウザのローカルに永続的に保存するバックグラウンド非同期スニッフィングアルゴリズムが組み込まれています。使用時間が長くなるにつれて、ローカルの再生時間データベースがより完全になり、重複排除エンジンが補完されたデータに基づいて、名前は異なるが内容の長さが同じである隠れた重複リソースをさらに見つけ出すことができるようになります。その結果、より多くの、より正確な結果が表示されるようになります。

**Q: 「画像検索」で画像を直接アップロードできず、貼り付けるように求められることがあるのはなぜですか？**  
**A:** これはブラウザのセキュリティポリシー（CORS クロスドメイン制限）によるものです。スクリプトがクラウドドライブの画像を画像検索サーバーに直接プッシュできない場合、現在の画面のスクリーンショットをクリップボードに自動的にコピーします。ポップアップ表示された検索ページで `[Ctrl+V]` を押すだけで、認識を完了できます。

**Q: スクリプトを使用してビデオを再生したときに、音だけが聞こえて画像が表示されないのはなぜですか？**  
**A:** これは通常、そのビデオが m3u8 マルチオーディオ/マルチトラック ストリーミングメディアに属しているためです。クラウドでのトランスコーディング後、オーディオと画像のトラックが分離されましたが、Webプレーヤーは現在、このような特殊なビデオストリームのオンラインマルチトラック解析をサポートしていません。右下隅の「外部再生」ボタンをクリックして、PotPlayerなどのローカルの専門的なプレーヤーを直接呼び出してストリーミング再生することをお勧めします。

**Q: 解像度メニューの「オリジナル (高速)」と通常の「オリジナル」の違いは何ですか？**  
**A:** 2つの解像度はほぼ完全に同じで、主な違いは基盤となるデコードと転送プロトコルにあります。「オリジナル (高速)」は、公式にストリーミングメディアに最適化されたリンクを使用しています。通常、PotPlayerなどの外部プレーヤーを呼び出す場合、「高速」という単語が含まれるリンクを使用すると、最速の読み込み応答と最もスムーズなドラッグエクスペリエンスが得られます。Web再生でも外部ストリーミングでも、このオプションを優先的に選択することを強くお勧めします。

**Q: モバイルアプリや公式Webで実行したばかりの操作（アップロード、削除、移動など）が、スクリプトの重複排除や分析リストにリアルタイムで同期されないのはなぜですか？**  
**A:** スクリプトは究極の検索と分析速度を保証するため、メモリスナップショットメカニズムを採用しており、毎回何万ものファイル情報を再取得することはありません。解決策：他のクライアントでファイルをごく多く変更した場合は、Webページを更新して全体のインデックスを再取得してください。スクリプトの操作中は、複数端末で同時にデータを変更しないようにしてください。

**Q: 一括解凍時、複数のファイルに一度に異なるパスワードを設定できますか？**  
**A:** 可能です。スマートマッチングロジック：既知のパスワードを事前に「パスワード保管庫」に保存しておけば、スクリプトは一括解凍時に各圧縮ファイルに対して保管庫内のすべてのパスワードを自動的に試します。一致した場合は手動で介入することなく自動的に開始されます。

**Q: 一括解凍時に「システムがビジー状態です/再試行を待機中」と表示された場合、スクリプトのバグですか？**  
**A:** いいえ。これは通常、PikPak クラウドサーバーが同時実行頻度制限をトリガーしたためです。このプロンプトが表示された場合、クラウドドライブの公式インターフェースも現在ビジー状態であることを意味します。複数回試行してもファイルが失敗する場合は、クラウド上のリソースが破損しているか制限されている可能性があるため、後で再試行するか、ローカルにダウンロードして処理することをお勧めします。

**Q: エクスポートされた設定JSONにはどのような情報が含まれていますか？**  
**A:** JSONファイルには、強化マスターで設定した重複排除の環境設定、ブラック/ホワイトリストのルール、パスワード保管庫などのローカル設定データのみが含まれ、クラウドドライブのアカウントパスワードは**含まれていません**。デバイス間の移行に安心して使用できます。

**Q: バックアップをインポートすると、既存の設定やリストが上書きされますか？**  
**A:** 上書きされません。スクリプトはスマートマージロジックにアップグレードされており、インポート時にリソースマネージャーや履歴などのリスト項目とローカルデータを自動的に重複排除してマージします。基本設定項目（検索エンジン、Aria2アドレスなど）のみがインポートされたファイルに依存するため、ローカルに蓄積されたデータが失われることはありません。

**Q: ファイルを一括削除するときに、一部のファイルが保護されているか削除できないというメッセージが表示されるのはなぜですか？**  
**A:** これらのファイルがリソースマネージャーに記録されているかどうかを確認してください。設定で「削除時にマネージャーに記録されているリソースをスキップする」を有効にしている場合、スクリプトは誤削除を防ぐために一致するファイルを保護された項目とみなします。<br>
解決策: 
1. 設定でこの保護ルールのチェックを外してから削除します。
2. または、ツールバー/サイドバーの「リソースマネージャー」エントリをクリックし、「今すぐクリーンアップを実行」をクリックして、これらの記録項目を物理的に強制削除します。

**Q: 「貼り付け」操作を実行した後、ファイルがリストに表示されないのはなぜですか？**  
**A:** クラウドドライブの空き容量が、これらのファイルを収容するのに十分かどうかを確認してください。PikPak公式は現在、「容量不足」の状況に対して**サイレントインターセプト (Silent Interception)** 戦略を採用しています。つまり、制限を超えた場合に具体的なエラーメッセージをポップアップ表示せず、バックグラウンドで操作を直接中止します。貼り付け後にファイルが表示されない場合は、通常、容量制限がトリガーされたためです。スペースを解放してから再試行することをお勧めします。

**Q: クラウドダウンロードポップアップの「ブロック防止マグネットリンクの自動修復」機能とは何ですか？いつオンにするのがおすすめですか？**  
**A:** この機能は、リンクに混在している漢字、絵文字、記号などのテキスト干渉を自動的に排除し、特徴コードをスマートに識別および抽出して、標準形式に復元することができます。<br>
**推奨: この機能はデフォルトでチェックを入れておくことをお勧めします。ソーシャルプラットフォームから取得した、無関係なテキストが大量に混ざっている、または「ヘッダーのない」汚染されたマグネットリンクを自動的に処理できます。`magnet:?` で始まる標準のマグネットリンクがすでに含まれている場合は、修復をスキップして正常にダウンロードします。**

**Q: Aria2/Motrix またはブラウザを使用してクラウドドライブのフォルダをダウンロードする場合、階層構造は保持されますか？**  
**A:** **Aria2/Motrix は階層を保持します**。スクリプトはディレクトリ階層のクローン技術を採用しており、ダウンロードが完了すると、ローカルハードドライブがクラウドドライブ内のツリー構造を完璧に復元します。さらに、クラウドドライブのフォルダ名からWindowsでサポートされていない特殊文字（`:` `*` `?` など）を自動的にクリーンアップし、ダウンロードエラーが発生しないように冗長なパスをスマートに短縮します。  
**ブラウザのダウンロードでは階層は保持されません**。ブラウザのネイティブダウンロードプロトコルの制限により、フォルダ内のすべてのファイルは同じディレクトリに「フラット」にダウンロードされます。**推奨:** 複雑なフォルダ構造を保持する必要がある場合は、必ず Aria2 プッシュを優先的に選択してください。

**Q: 「マルチアカウントデータ移行」とは何ですか？具体的にどのように操作しますか？**  
**A:** この機能を使用すると、現在のアカウントのファイルを別のアカウントにシームレスかつ迅速に転送できます。
1. 現在のアカウントで転送するファイルまたはフォルダを選択し、下部の「データ移行」ボタンをクリックします。
2. スクリプトは自動的にデータを暗号化してパッケージ化し、現在のアカウントから自動的にログアウトします。
3. その後、[ターゲットアカウント] に通常通りログインするだけで、ログイン成功後にスクリプトがローカルにキャッシュされた移行パッケージを自動検出し、ワンクリックで受信するようにポップアップメッセージを表示します。

## 🛡️ プライバシーとセキュリティに関する声明 (Privacy & Security)

* **データのローカル化**: このスクリプトのすべてのコア機能（一括リネーム、ファイル分析、解凍など）は、ブラウザを介して PikPak 公式 API と直接対話します。アカウント トークン、パスワード保管庫、およびファイル データは、**ローカルブラウザにのみ保存されます**。
* **収集ゼロ**: スクリプトはユーザーのプライバシーデータを**収集せず**、ファイル情報やアカウント資格情報をサードパーティのサーバーにアップロードすることは**決してありません**。
* **サードパーティのインターフェース**: 「オンライン字幕検索」または「画像検索」などの拡張機能を使用する場合にのみ、スクリプトは関連する公共サービスインターフェースに必要な検索キーワードまたは画像特性パラメーターを送信し、個人の識別情報には関与しません。

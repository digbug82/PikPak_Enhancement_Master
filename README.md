## ❓ 常见问题解答 (FAQ)

**Q：为什么安装了脚本却没有显示蓝色的悬浮球？**  
**A：** 请确保您已经登录了 PikPak 网页端。如果已登录仍未显示，请尝试刷新网页 (F5)；或者检查是否被广告拦截插件（如 AdGuard / uBlock Origin）误拦截，尝试暂时关闭拦截插件或将本站加入白名单。

**Q：什么是“极速模式 (Turbo Mode)”，为什么建议开启？**  
**A：** 开启后脚本将全屏覆盖网页端，并全面接管所有原生网页端功能。该模式通过物理屏蔽原网页极度消耗内存的同步逻辑，彻底解决海量文件下的卡顿与崩溃问题。**注意：** 极速模式下最小化与关闭按钮将自动隐藏（需在设置中取消勾选方可返回原网页）。**建议：** 若您的网盘文件较多，强烈推荐开启此模式，以获得极致的响应速度与流畅体验。

**Q: 为什么首页的 My Pack 显示“默认”标签且无法添加星标、复制、移动、重命名或删除？**  
**A:** 这是官方的安全保护机制。脚本在此基础上解除了默认文件夹 My Pack 的分享限制。

**Q：脚本中的“文件查重”与“文件夹查重”有什么本质区别？在实际使用中该如何选择？**  
**A：** **文件查重：** 针对“单一文件”级别。采用哈希精准匹配、视频时长相似度、名称相似度三模态算法，适合清理网盘中零散分布的重复文件（如重复保存的单集视频、单张图片等）。<br>
**文件夹查重：** 针对“文件夹整体结构”级别。采用名称匹配、内部文件重合度（相似度匹配）、子集冗余（包含率匹配）三模态算法。它能看透文件夹的“内部本质”，非常适合用来清理重复保存的“整部剧集”、“完整图包”或“多层级资料”，即使两个文件夹名字完全不同，只要内部文件高度一致或存在包含关系，也能被精准揪出。

> **建议：** 文件查重中的精准查重为哈希匹配，可一键勾选删除。文件查重中的时长与名称匹配，以及文件夹查重三模态均为相似匹配，请谨慎一键勾选以防误删。

**Q: 为什么文件查重查出来的重复文件越来越多了？是不是全盘索引丢包了？**  
**A:** 请放心，这并非全盘索引丢包或数据错误，查重结果中“精准匹配”部分数量始终是稳定的。查重结果增多是因为基于视频时长的“相似匹配”变得更全面了：由于 PikPak 部分原始数据接口不直接提供视频时长，导致许多重复视频在初期因数据缺失而被跳过，为此脚本内置了后台异步嗅探算法，会自动模拟读取视频元数据来提取精确时长并持久化保存在浏览器本地。随着您使用时间的增加，本地时长数据库越来越完备，查重引擎得以基于补全后的数据揪出更多文件名不同但内容长度一致的隐藏重复资源，因此结果会显得越来越多、越来越准。

**Q: 为什么“以图搜图”有时无法直接上传图片，而是提示让我粘贴？**  
**A:** 这是受限于浏览器的安全策略（CORS 跨域限制）。当脚本无法直接将网盘图片推送到搜图服务器时，会自动将当前画面截图拷贝到您的剪贴板。您只需在弹出的搜图页面中按下 `[Ctrl+V]` 即可完成识别。

**Q: 为什么使用脚本播放视频时，只有声音却没有画面？**  
**A:** 这通常是因为该视频属于 m3u8 多音轨/多轨道流媒体，云端转码后声音与画面轨道发生了分离，而网页端播放器暂不支持此类特殊视频流的在线多轨解析。建议点击右下角的“外部播放”按钮，直接调用本地的 PotPlayer 等专业播放器进行串流播放。

**Q：清晰度菜单中的“原画（高速）”与普通“原画”有什么区别？**  
**A：** 两者清晰度几乎完全一致，主要区别在于底层的解码与传输协议不同。“原画（高速）”采用了官方专为流媒体优化的链路，通常在调用 PotPlayer 等外部播放器时，使用带“高速”字样的链接能获得最快的加载响应和最顺滑的拖动体验。无论是网页播放还是外部串流，均建议优先选择此选项。

**Q: 为什么我在手机 App 或官方网页版刚进行的操作（如上传、删除、移动），在脚本的查重或分析列表中没有实时同步？**  
**A:** 脚本为保证极致的搜索和分析速度，采用了内存快照机制，不会每次都重新拉取数万个文件信息。解决方法：若在其他客户端修改文件过多，请刷新网页重新获取全盘索引。在脚本操作期间请避免多端同时修改数据。

**Q: 批量解压时，可以一次性给多个文件设置不同的密码吗？**  
**A:** 可以。智能匹配逻辑： 只要您将已知的密码预先存入“密码金库”，脚本在批量解压时会针对每一个压缩包自动尝试金库中的所有密码。如果匹配成功则自动开始，无需人工干预。

**Q：批量解压时提示“系统繁忙/等待重试”是脚本出 Bug 了吗？**  
**A：** 不是。这通常是由于 PikPak 云端服务器触发了并发频率限制。出现此提示意味着网盘官方接口当前也处于繁忙状态。若文件多次尝试依然失败，说明该资源在云端可能已损坏或受限，建议稍后重试，或尝试重新上传、下载到本地处理。

**Q: 导出的配置 JSON 包含哪些信息？**  
**A:** JSON 文件仅包含您在增强大师中设置的查重偏好、黑白名单规则、密码金库等本地配置数据，**不包含**您的网盘账号密码，可放心用于跨设备迁移。

**Q：导入备份会覆盖我现有的配置或名单吗？**  
**A：** 不会。脚本已升级为智能合并逻辑，导入时会自动将资源管理器、历史记录等列表项与本地数据去重合并，仅基础设置项（如搜图引擎、Aria2 地址等）会以导入的文件为准，确保您的本地积累数据不会丢失。

**Q: 为什么批量删除文件时，部分文件提示受保护或无法被删除？**  
**A:** 请检查这些文件是否已被记录在资源管理器中。若您在设置中开启了“删除时跳过管理器中记录资源”，脚本会将匹配的文件视为受保护项目以防止误删。<br>
解决方法： 
1. 在设置中取消勾选该保护规则后再进行删除；
2. 或点击工具栏/侧边栏的“资源管理器”入口，点击“立即运行清理”来强制物理删除这些记录项。

**Q：为什么我执行“粘贴”操作后，文件没有在列表中显示？**  
**A：** 请检查您的网盘剩余空间是否足以容纳这些文件。PikPak 官方目前对“空间不足”的情况采取**静默拦截**策略，即在超出限额时不会弹出具体的错误提示，而是直接在后台中止操作。若粘贴后文件未出现，通常是因为触发了容量限制，建议清理空间后重试。

**Q：云下载弹窗中“自动修复防屏蔽磁链”是什么功能，建议何时开启？**  
**A：** 该功能可自动剔除链接中夹杂的汉字、表情、符号等文字干扰，并智能识别提取特征码，将其还原为标准格式。<br>
**建议：该功能默认保持勾选，可自动处理从社交平台获取的、夹杂大量无关文字或“去头”的污染磁链，已包含 `magnet:?` 开头的**标准磁链**会跳过修复正常下载。

**Q：使用 Aria2/Motrix 或浏览器下载网盘文件夹时，层级结构会保留吗？**  
**A：** **Aria2/Motrix 会保留层级**。脚本采用了目录层级克隆技术，下载完成后您的本地硬盘将完美还原网盘中的树状结构，且会自动清洗网盘文件夹名称中 Windows 不支持的特殊字符（如 `:` `*` `?` 等）并智能缩短冗余路径以确保下载不报错。  
**浏览器下载则不会保留层级**。受限于浏览器原生下载协议的局限，文件夹内的所有文件将被“平铺”下载到同一个目录下。**建议：** 若需保留复杂的文件夹结构，请务必优先选择 Aria2 推送。

**Q：什么是“多账号数据迁移”？具体该如何操作？**  
**A：** 该功能允许您将当前账号的文件无缝、快速地转存到您的另一个账号。
1. 在当前账号选中需要转移的文件或文件夹，点击底部的“数据迁移”按钮。
2. 脚本会自动将数据加密打包，并自动为您退出当前账号。
3. 随后，您只需正常登录您的【目标账号】，脚本会在登录成功后自动侦测到本地缓存的迁移包，并弹窗提示您一键接收。

## 🛡️ 隐私与安全声明 (Privacy & Security)

* **数据本地化**：本脚本所有核心功能（如批量重命名、文件分析、解压等）均通过您的浏览器直接与 PikPak 官方 API 进行交互。您的账号 Token、密码金库及文件数据均**仅保存在本地浏览器中**。
* **零收集**：脚本**不会**收集任何用户隐私数据，也**绝不会**将您的文件信息或账号凭证上传至任何第三方服务器。
* **第三方接口**：仅在使用“在线字幕搜索”或“以图搜图”等扩展功能时，脚本会向相关公共服务接口发送必要的搜索关键词或图片特征参数，不涉及您的任何个人身份信息。

*  对登录闪烁问题进一步优化，若发现仍存在脚本一直卡在“正在同步登录状态”请及时反馈您的浏览器版本。

### V1.3.0
* **核心更新**：新增**多账号数据迁移**功能，支持一键将海量文件加密打包并无缝转存至您的其他账号。
* **核心更新**：新增**彻底删除**机制，删除时可一键跳过回收站，瞬间释放网盘空间。
* **UI 与交互体验优化**：
  * 修复了图片在极端缩放或长图模式下，无法拖拽至图片最边缘的边界限制计算错误问题。
  * 全面修复了“最近添加”、“分析”等特殊视图下，路径栏下拉文件夹列表排序与表头不一致、排序模式失效等问题。
  * 优化了多尺寸屏幕（高低分辨率）的响应式布局切换阈值与交互细节。
  * 修复了某些特殊情况下模态窗口层级相互遮挡的问题。
  * 强化了未登录状态的侦测逻辑，避免极端情况下的网络死循环重试。

### V1.2.0
*   **视频时长嗅探引擎重构**：深度加固视频时长嗅探逻辑。
*   **UI 交互优化**：精修部分 UI 组件在不同缩放比例下的视觉表现。
*   **核心 Bug 修复**：修复路径检索时高亮失效的 Bug；修复在设置中保存“极速模式”修改时会导致其他设置项被阻断失效的 Bug。

### V1.1.0
*   **Aria2 增强**：支持**保留文件夹路径结构**，新增投递丢包提醒及错误清单一键导出。
*   **云下载进化**：支持批量提交与智能去重，**自动清洗防屏蔽污染磁链**（支持 Base32 自动解码）。
*   **重命名优化**：预览界面新增**图标与封面展示**，支持文件夹悬浮大图预览。
*   **配置管理**：导入功能升级为**“智能合并”**模式，导入备份时不再覆盖本地已有的名单和记录。
*   **体验提升**：大幅优化极速模式加载速度，修复账号切换时的 UI 残留 Bug，提升搜图成功率。
